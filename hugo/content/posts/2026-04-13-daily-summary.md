# Daily Threat Intel Digest - 2026-04-13

## 🔴 Critical Threats & Active Exploitation

**[NEW] AI-Assisted Attack Framework Automates Large-Scale Government Intrusions**
A newly released technical report confirms artificial intelligence is being actively weaponized in advanced cyberattacks against Mexican government agencies. Forensic analysis uncovered a 17,550-line Python management script that piped server telemetry into OpenAI's API, processing data from 305 internal servers into 2,597 structured "intelligence reports." The attacker rapidly identified exploitable targets using synthesized output and deployed 400 unique scripts, including 20 adapted exploits for 20 different CVEs. Metadata from 34 live sessions revealed 1,088 logged prompts generating over 5,300 AI-executed commands. By fusing generative AI with offensive tradecraft, attackers can now scale operations with minimal human labor—effectively democratizing capabilities once reserved for state-sponsored groups. Gaps in basic cyber hygiene (unpatched software, static credentials, absent segmentation) created the staging ground for automation-driven attacks. [[CyberPress](https://cyberpress.org/ai-tools-target-agencies/)] [[Gambit Security Technical Report](https://gambit.security/blog-post/a-single-operator-two-ai-platforms-nine-government-agencies-the-full-technical-report)]

**[NEW] Marimo Notebook RCE Exploited Within 10 Hours of Disclosure**
A critical remote code execution vulnerability in the open-source Marimo reactive Python notebook platform (CVE-2026-39987, CVSS 9.3) was actively exploited less than 10 hours after public disclosure. The flaw stems from the `/terminal/ws` WebSocket endpoint exposing an interactive terminal without proper authentication, granting unauthenticated attackers full interactive shell access running with Marimo process privileges. Within 12 hours of disclosure, 125 IP addresses began reconnaissance activity. Attackers validated the vulnerability, executed basic environment reconnaissance, and completed credential harvesting—including .env files with cloud credentials and SSH keys—in under three minutes. A second exploitation session occurred approximately one hour later. The operator demonstrated methodical hands-on tradecraft focused on high-value objectives rather than automated scripts. Marimo users should immediately upgrade to version 0.23.0, block or disable the `/terminal/ws` endpoint, restrict external access via firewall, and rotate all exposed secrets. [[BleepingComputer](https://www.bleepingcomputer.com/news/security/critical-marimo-pre-auth-rce-flaw-now-under-active-exploitation/)] [[Sysdig Research](https://www.sysdig.com/blog/marimo-oss-python-notebook-rce-from-disclosure-to-exploitation-in-under-10-hours)]

**[NEW] Adobe Acrobat Reader Zero-Day Actively Exploited in Wild**
Adobe released an emergency security update (APSB26-43) patching a critical zero-day vulnerability (CVE-2026-34621) in Acrobat and Acrobat Reader for Windows and macOS that is being actively exploited. The flaw, caused by improper control of object prototype attributes (Prototype Pollution), allows arbitrary code execution via maliciously crafted PDF documents. With a CVSS score of 8.6 (CVSS:3.1/AV:L/AC:L/PR:N/UI:R/S:C/C:H/I:H/A:H), local attack vectors combined with minimal user interaction suffice for exploitation. Affected versions include Acrobat DC Continuous Track ≤26.001.21367 and Acrobat 2024 Classic Track ≤24.001.30356. Adobe classifies this as Priority 1, requiring immediate deployment. Security researcher Haifei Li from EXPMON is credited with discovery. [[CyberPress](https://cyberpress.org/adobe-patches-acrobat-reader-zero-day-vulnerability/)] [[Adobe APSB26-43](https://helpx.adobe.com/security/products/acrobat/apsb26-43.html)]

**[NEW] Storm-2755 "Payroll Pirate" Campaign Hijacks Canadian Employee Salaries**
Microsoft's Detection and Response Team (DART) is tracking Storm-2755, a financially motivated threat actor running payroll fraud attacks that quietly redirect Canadian employees' salaries to attacker-controlled bank accounts. The campaign begins with SEO poisoning and malvertising pushing an attacker-controlled domain (bluegraintours[.]com) to the top of search results for generic queries like "Office 365" or misspellings like "Office 265." Users are funneled to a counterfeit Microsoft 365 sign-in page implementing adversary-in-the-middle (AiTM) authentication proxying, capturing both credentials and live session tokens in real time. The operation exploits CVE-2025-27152 in Axios (HTTP client version 1.7.9) for SSRF-based pivoting during token replay. Storm-2755 harvests session cookies and OAuth access tokens that bypass non-phishing-resistant MFA flows, maintaining session validity for up to 30 days through periodic non-interactive sign-ins every 30 minutes. Microsoft recommends treating this as a blueprint for salary-fraud operations built around session hijacking: revoke active tokens, remove malicious inbox rules, reset passwords and MFA methods, and prioritize phishing-resistant MFA (FIDO2/WebAuthn) with Conditional Access session-lifetime controls. [[CyberPress](https://cyberpress.org/storm-2755-hijacks-employee-salaries/)] [[Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/04/09/investigating-storm-2755-payroll-pirate-attacks-targeting-canadian-employees/)]

## 🎯 Threat Actor Activity & Campaigns

**[NEW] APT37 Pretexting Campaign Abuses Facebook, Telegram, and Trojanzied Installers**
North Korean threat actor APT37 is running a targeted intrusion campaign abusing social media platforms and supply chain compromise to target defense-related organizations. The operation begins with reconnaissance using Facebook accounts claiming to be from Pyongyang and Pyongsong, building trust by adding targets as friends before moving conversations to Messenger. Targets are lured with specific topics related to military weapons and tricked into installing a dedicated PDF viewer under the pretense of sharing an encrypted document. The initial access vector is a tampered Wondershare PDFelement installer that executes shellcode, followed by JPG-disguised payloads delivered via a compromised Japanese real estate information service website's Seoul branch. Behavior-based EDR is required to detect process injection, abused cloud storage, and APT evasion techniques. [[GBHackers](https://gbhackers.com/new-targeted-cyberattack/)] [[Genians Threat Intelligence](https://www.genians.co.kr/en/blog/threat_intelligence/pretexting)]

**[UPDATE] Rockwell PLC Exposure Expands—New Attacker Infrastructure Identified**
Following last week's joint advisory (AA26-097A) about Iranian-linked hackers targeting Rockwell Automation PLCs, new scanning data from Censys reveals over 5,200 exposed industrial devices currently accessible on the public internet, with 75% hosted in the United States. Notably, 50% of global exposures rely on Verizon Business cellular networks, indicating field-deployed devices such as water pumps or energy substations connected via basic cellular modems. Deeper analysis of attacker infrastructure reveals that the seven IP addresses initially flagged by government agencies actually belong to a single Windows engineering workstation operated by the attackers, loaded with Rockwell's own engineering tools and serving as the attacker's personal launchpad. By tracking a unique remote desktop security certificate tied to the name "DESKTOP-BOE5MUC," researchers identified four additional attacker IP addresses missing from the official advisory. Another listed IP was identified as a Romanian "burner" staging server, freshly rented for scanning in mid-March then abandoned. Organizations must remove PLCs from direct internet exposure, use secure gateway MFA for remote access, set CompactLogix and MicroLogix physical key switches to "RUN" position to prevent remote programming changes, and block the full range of attacker IP addresses. [[CyberPress](https://cyberpress.org/rockwell-plc-exposure-grows/)] [[Censys Research](https://censys.com/blog/iranian-affiliated-apt-targeting-rockwell-allen-bradley-plcs/)]

## ⚠️ Vulnerabilities & Patches

**[NEW] WordPress User Registration Plugin Authentication Bypass—Admin Takeover Possible**
A critical vulnerability (CVE-2026-1492) in the User Registration & Membership plugin for WordPress allows unauthenticated attackers to bypass authentication and gain full administrative control. The flaw stems from improper trust handling between frontend and backend components, leaving internal tokens exposed in client-side JavaScript contexts. Attackers can reuse valid security tokens to craft AJAX requests targeting `/wp-admin/admin-ajax.php` that execute administrative actions—including creating accounts with elevated privileges—without login credentials. Once inside, adversaries can install or modify plugins and themes, execute arbitrary code, or establish persistence via hidden admin accounts. Testing confirmed multiple exploitation paths through role parameter manipulation during registration and membership workflow abuse. Affected plugin versions include User Registration & Membership ≤5.1.2; the vendor patched this in v5.1.3. Administrators should immediately upgrade, implement strict role-based access rules, and monitor for suspicious POST activity to AJAX endpoints. [[CyberPress](https://cyberpress.org/wordpress-bug-enables-takeover/)] [[CYFIRMA Research](https://www.cyfirma.com/research/cve-2026-1492-wordpress-user-registration-membership-authentication-bypass-flaw/)]
