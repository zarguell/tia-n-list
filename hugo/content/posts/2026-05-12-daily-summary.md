## 🔴 Critical Threats & Active Exploitation

**[NEW] Google detects first confirmed AI-generated zero-day exploit — threat actors industrializing vulnerability discovery**

Google's Threat Intelligence Group (GTIG) has confirmed the first documented case of a threat actor developing a functional zero-day exploit using AI. The exploit targeted a popular open-source web-based administration tool, bypassing its two-factor authentication by exploiting a hardcoded trust assumption in a Python script — a high-level semantic logic bug that AI systems excel at identifying, distinct from memory corruption or fuzzing-discovered flaws. GTIG discovered the exploit before mass exploitation began and notified the vendor, who patched it. The exploit code carried unmistakable AI fingerprints: extensive educational docstrings, highly structured Pythonic formatting, and a hallucinated CVSS severity score the attackers forgot to remove. The responsible cybercrime group has a "strong record of high-profile incidents and mass exploitation." Google is confident the AI model used was neither Gemini nor Anthropic's Mythos. Beyond this single case, GTIG documents that Chinese APT groups (APT27, UNC2814, UNC5673) are using expert persona prompts to bypass AI safety filters and analyze router firmware for RCE flaws, North Korean actors (APT45) are automating recursive CVE analysis at machine speed, and Russia-linked groups are using AI-generated decoy code (CANFAIL, LONGSTREAM, HONESTCUE) to blind security analysts. The Android backdoor PROMPTSPY directly integrates with the Gemini API to read device interfaces, interpret system states, and simulate physical taps to capture biometric data. **[NEW] BitUnlocker bypasses Windows 11 BitLocker in under 5 minutes — CVE-2025-48804 downgrade attack**

A proof-of-concept tool called BitUnlocker demonstrates a downgrade attack that bypasses BitLocker full-disk encryption on fully patched Windows 11 machines with only physical access. The attack exploits CVE-2025-48804, chaining a boot manager downgrade with a manipulated Windows Recovery Environment (WinRE) to silently decrypt BitLocker-protected drives in under five minutes. Because Microsoft's July 2025 patch only updated `bootmgfw.efi` to the CA 2023 certificate, any older version signed under the still-trusted Microsoft Windows PCA 2011 certificate can be substituted. The attacker boots a custom WinRE image via USB or PXE, loads a pre-patch boot manager, and reaches a decrypted command prompt — no BitLocker PIN or recovery key required. TPM-only and PCR 7+11 configurations (the most common enterprise defaults) are vulnerable. Systems using TPM+PIN, the Windows UEFI CA 2023 certificate, or DBX revocation are protected. The PoC is publicly available on GitHub under MIT license, making it trivially reproducible. Organizations should enable TPM+PIN pre-boot authentication immediately and verify their boot manager certificate via `sigcheck -i`. [[Cyber Security News](https://cyberpress.org/new-bitunlocker-attack-bypasses-windows-11-disk-encryption-in-just-5-minutes/); [GBHackers](https://gbhackers.com/bitunlocker-downgrade-attack-bypasses-windows-11-disk-encryption/)]


## 🎯 Threat Actor Activity & Campaigns

**[NEW] TeamPCP supply chain campaign escalates — 84 TanStack npm packages, MistralAI PyPI, Checkmarx Jenkins all compromised in cascading attacks**

TeamPCP's ongoing supply chain campaign has expanded to three new fronts in a single 24-hour cycle. First, 84 npm packages in the TanStack ecosystem — including React Router with 12 million weekly downloads — were compromised with a credential-stealing worm (`router_init.js`) that targets GitHub Actions tokens, AWS credentials, Kubernetes certificates, and HashiCorp Vault clusters. The malware uses the Session P2P network for exfiltration and writes persistence hooks into Claude Code and VS Code configuration directories. TanStack maintainers attributed the breach to a `pull_request_target` workflow exploit that extracted OpenID Connect tokens from runner process memory. Second, version 2.4.6 of the official `mistralai` PyPI package was backdoored to execute on import, downloading a secondary payload from `83[.]142[.]209[.]194` that functions as a credential stealer on Linux systems. The payload includes geofenced destructive logic — a 1-in-6 probability of `rm -rf /` on systems identified as Israeli or Iranian — while skipping Russian-language environments entirely. Microsoft is investigating. Third, a rogue version (2026.5.09) of the Checkmarx Jenkins AST plugin was pushed to the Jenkins Marketplace using credentials stolen from the March Trivy breach — the third supply chain incident affecting Checkmarx since late March. The attackers left a taunting message: "Checkmarx fails to rotate secrets again." Last known-good version: 2.0.13-829.vc72453fa_1c16 (December 17, 2025). [[Cyber Security News](https://cyberpress.org/84-tanstack-npm-packages-compromised-in-ongoing-supply-chain-attack-targeting-ci-credentials/); [GBHackers](https://gbhackers.com/mistralai-pypi-package/); [BleepingComputer](https://www.bleepingcomputer.com/news/security/official-checkmarx-jenkins-package-compromised-with-infostealer/)]

**[NEW] DFIR Report documents The Gentlemen ransomware — EtherRAT via blockchain C2, AI-generated TukTuk framework, SaaS-based command infrastructure**

The DFIR Report has published a flash alert documenting a complete intrusion chain ending in The Gentlemen ransomware deployment. Initial access came via a malicious MSI masquerading as Sysinternals RAMMap, which deployed EtherRAT — a malware family that uses the Ethereum blockchain via EtherHiding to dynamically update its C2 configuration by querying smart contracts through 1rpc.io. The threat actor later deployed TukTuk, a newly documented malware framework believed to be AI-generated, using DLL sideloading through trojanized Greenshot, SyncTrayzor, DocFX, and Cake binaries. TukTuk's C2 infrastructure is built entirely on SaaS platforms — ClickHouse, Supabase, Ably, Dropbox, GitHub Issues — with Arweave blockchain dead-drop resolution as a fallback. The intrusion progressed through Kerberoasting, LSASS/NTDS credential dumping via comsvcs.dll, lateral movement using GoTo Resolve RMM, and data exfiltration to Wasabi cloud storage via Rclone, culminating in domain-wide ransomware deployment through a malicious Group Policy Object. The heavy use of blockchain and SaaS infrastructure makes this campaign resilient to traditional network defenses — defenders should monitor for unexpected outbound connections to Ethereum RPC endpoints, Arweave gateways, and SaaS platforms not approved in their environment. [[The DFIR Report](https://thedfirreport.com/2026/05/11/flash-alert-etherrat-and-tuktuk-c2-end-in-the-gentleman-ransomware/)]


## ⚠️ Vulnerabilities & Patches

**[UPDATE] cPanel CVE-2026-41940 — Mr_Rot13 syndicate deploys Go-based infector with credential-harvesting JavaScript injection**

Detailed technical analysis of the ongoing cPanel/WHM exploitation campaign reveals a sophisticated operation by the Mr_Rot13 group, active for over six years with exceptionally low detection rates. Beyond the initial CVE-2026-41940 (CVSS 9.8) authentication bypass, the attackers deploy a custom Go-based payload infector that modifies root passwords, implants SSH public keys labeled `cpanel-updater`, drops a Python webshell, and injects malicious JavaScript into cPanel login interfaces to harvest admin credentials in real-time. Stolen data is exfiltrated via ROT13-obfuscated domains and Telegram channels. A cross-platform remote control Trojan (`filemanager`) provides persistent web-based GUI access with bcrypt authentication. The group's infrastructure links back to a 2022 PHP backdoor targeting WordPress that maintained zero detection for years. Ctrl-Alt-Intel reported that the campaign was used to infiltrate Southeast Asian government and military networks, exfiltrating 4 GB of sensitive data. [[GBHackers](https://gbhackers.com/cpanel-and-whm-servers-targeted/); [Cyber Security News](https://cyberpress.org/hackers-exploit-cve-2026-41940/)]

**[NEW] Critical PHP SOAP extension RCE — CVE-2026-6722 enables remote code execution via crafted XML payloads**

A high-severity Use-After-Free vulnerability (CVE-2026-6722) in PHP's SOAP extension allows remote attackers to achieve code execution by exploiting improper reference counting during XML graph deduplication. The extension stores plain objects in a hash map using `id` and `href` attributes but fails to increment reference counts, allowing attackers to force premature object freeing via Apache map mechanisms, manipulate freed memory through string allocation, and achieve reliable RCE. Additional vulnerabilities include CVE-2026-7261 (UAF in SoapServer session persistence), CVE-2026-7262 (NULL pointer dereference causing DoS), and CVE-2026-7258/CVE-2026-6104 (out-of-bounds reads in PHP core). Affected versions: PHP before 8.2.31, 8.3.31, 8.4.21, and 8.5.6. Administrators running PHP applications that process untrusted XML should upgrade immediately. [[Cyber Security News](https://cyberpress.org/critical-php-soap-extension-flaw/); [GBHackers](https://gbhackers.com/php-soap-extension-flaw/)]


## 🛡️ Defense & Detection

**[NEW] Hugging Face repository with 200K downloads distributes Windows infostealer — popularity artificially inflated**

Security researchers have identified the Hugging Face repository `Open-OSS/privacy-filter` — which amassed over 200,000 downloads — as a conduit for a sophisticated credential-harvesting infostealer. The threat actors artificially inflated the repository's popularity using fake accounts to reach the top trending spot. The attack chain: a model card copied from a legitimate project instructs users to run scripts that disable SSL verification, fetch commands from a public JSON paste service for updatable payloads, and execute hidden PowerShell targeting Windows systems. A second-stage downloader mimics a blockchain analytics API, checks for admin privileges, downloads the final payload, and adds Microsoft Defender exclusions. Related repositories under the account `anthfu` contain identical loader scripts, and the infrastructure shares domains with previously documented npm typosquatting operations, indicating a coordinated supply chain operation targeting open-source ecosystems. [[Cyber Security News](https://cyberpress.org/hugging-face-repo-spreads-malware/)]

**[NEW] Fake Claude website deploys Beagle backdoor via DLL sideloading — also mimics Trellix, CrowdStrike, SentinelOne**

A fake website at `claude-pro[.]com` distributes a trojanized Windows installer that deploys a newly documented backdoor called Beagle through a DLL sideloading chain. The attack abuses a signed G DATA updater executable (`NOVupdate.exe`) to load a malicious `avk.dll`, which decrypts shellcode via DonutLoader. The installer also provides a working copy of Claude as camouflage. Sophos found additional 2026 samples on VirusTotal using the same XOR key, including one linked to the AdaptixC2 framework and others themed around Trellix, CrowdStrike, and SentinelOne brands — suggesting the operators are experimenting with multiple fake-brand lures.
