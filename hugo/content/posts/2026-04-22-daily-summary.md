# Daily Threat Intel Digest - April 22, 2026

## 🔴 Critical Threats & Active Exploitation

**[NEW] Self-propagating npm supply-chain worm steals credentials across ecosystems**
A sophisticated supply-chain attack targeting the Namastex npm ecosystem is stealing developer credentials and autonomously spreading to new packages, exhibiting worm-like behavior that security researchers have linked to the TeamPCP threat group's CanisterWorm operations. The malware searches for `.npmrc` files, SSH keys, cloud credentials, and cryptocurrency wallet data, then uses stolen npm publishing tokens to inject malicious code into other packages the victim can publish—creating recursive compromise across the dependency tree. Affected packages include `@automagik/genie`, `pgserve`, `@fairwords/websocket`, and `@openwebconcept` variants used in AI agent tooling and database operations. The payload also attempts to spread to PyPI via `.pth` techniques, making this a multi-ecosystem threat. Socket's analysis confirmed 16 compromised package versions with meaningful weekly download volumes, putting active development environments at immediate risk [[BleepingComputer](https://www.bleepingcomputer.com/news/security/new-npm-supply-chain-attack-self-spreads-to-steal-auth-tokens/); [Cyberpress](https://cyberpress.org/namastex-packages-drop-canisterworm/)]. Organizations should remove affected versions immediately, rotate all exposed secrets, and audit for the associated public.pem file and webhook infrastructure.

**[NEW] Malicious Google Ads deploy wallet drainers through cloned DeFi interfaces**
A sustained campaign tracked by the Security Alliance (SEAL) is weaponizing Google Ads against cryptocurrency users by cloning legitimate DeFi applications and wallet services to steal seed phrases and drain wallets through drainer-as-a-service (DaaS) platforms like Inferno Drainer and Vanilla Drainer. The operation exploits hacked and purchased verified advertiser accounts to distribute ads via high-reputation Google URLs (sites.google.com, docs.google.com), then uses Arweave decentralized storage and Cloudflare Workers to serve obfuscated 2.7 MB payloads that evade automated scanners. A three-component architecture includes a man-in-the-middle proxy layer that monkey-patches `window.fetch` and `XMLHttpRequest`, routing all Ethereum RPC calls through attacker-controlled infrastructure for real-time transaction monitoring. Over 356 malicious ad URLs were blocked in a three-week window, with campaigns sustained for over a year and currently escalating [[GBHackers](https://gbhackers.com/google-ads-hit-crypto-users/); [Cyberpress](https://cyberpress.org/fake-ads-drain-wallets/)]. The attack targets users across MetaMask, Exodus, Atomic Wallet, and Phantom.

**[NEW] Lotus Wiper destroys Venezuelan energy sector systems with no recovery option**
A destructive wiper campaign targeting Venezuela's energy and utilities sector is permanently destroying data across physical drives using a new malware family dubbed Lotus Wiper, which leaves systems completely unrecoverable by overwriting drives with zeroes, clearing USN journals, and deleting all Windows restore points. The attack chain begins with batch scripts (`OhSyncNow.bat`, `notesreg.bat`) that use a NETLOGON share XML file as a network trigger before resetting all local passwords to random strings, disabling cached logins, and forcibly logging off users. Kaspersky researchers identified artifacts uploaded to public malware repositories in mid-December 2025, with compilation timestamps indicating the wiper was built in late September 2025 during heightened geopolitical tensions. The malware systematically wipes both physical drives and mounted volumes using IOCTL disk geometry calls and `FSCTL_SET_ZERO_DATA` to destroy sector-level data before final deletion [[GBHackers](https://gbhackers.com/lotus-wiper/); [Cyberpress](https://cyberpress.org/lotus-wiper-hits-energy/)]. Unlike ransomware, this attack carries no extortion component—its sole purpose is annihilation.

**[NEW] Atlassian Bamboo critical flaw enables remote command injection in CI/CD pipelines**
A critical OS command injection vulnerability (CVE-2026-21571, CVSS 9.4) in Atlassian Bamboo Data Center and Server allows authenticated attackers to execute arbitrary commands on affected systems, posing severe supply-chain risks for organizations using the CI/CD platform. The flaw requires only low-level authentication with no user interaction, enabling attackers to inject malicious code into automated build processes and potentially compromise downstream software deployments. Atlassian has released patches for Bamboo versions 9.6.25, 10.2.18, and 12.1.6 to address the vulnerability, which affects multiple major versions including 9.6.0, 10.0.0 through 12.1.0. This is one of 38 vulnerabilities in Atlassian's April 2026 Security Bulletin, including a CVSS 10.0 XSS flaw and additional RCE issues tied to third-party dependencies [[Cyberpress](https://cyberpress.org/critical-bamboo-data-center-and-server-vulnerability-enables-command-injection-attacks/); [GBHackers](https://gbhackers.com/critical-bamboo-data-center-and-server-flaw/)]. Organizations running Bamboo should upgrade immediately and audit CI/CD pipelines for unauthorized modifications.

## 🎯 Threat Actor Activity & Campaigns

**[NEW] Iranian MOIS-linked actors adopt Russian MaaS tools for espionage campaigns**
Iran's MuddyWater espionage group (MOIS-aligned) is now using the Russian-developed CastleRAT malware-as-a-service platform alongside a newly discovered Node.js agent called ChainShell, marking a convergence of state-sponsored espionage and criminal MaaS ecosystems that complicates attribution. A misconfigured command-and-control server exposed Farsi code comments, Israeli target IP lists, and campaign identifiers linking the same operators to both CastleRAT payloads and known MuddyWater tools like StageComp—connected via shared code-signing certificates issued to "Amy Cherne." CastleRAT provides keylogging, Chrome cookie decryption, and Hidden VNC capabilities, while ChainShell resolves C2 addresses through the Ethereum blockchain for infrastructure resilience. The malware contains Russian-language strings and avoids infecting post-Soviet systems, potentially causing defenders to misattribute campaigns to Russian cybercriminals rather than Iranian state actors [[Cyberpress](https://cyberpress.org/iran-tied-to-castlerat/)]. Security teams should escalate incidents showing this tooling as potential state-level espionage rather than commodity crime.

**[NEW] Harvester APT expands to Linux with GoGra backdoor using Microsoft Graph API**
The Harvester threat group, active since at least 2021 and known for targeting telecommunications and government organizations in South Asia, has expanded its toolset with a Linux variant of the GoGra backdoor that uses Microsoft Graph API and Outlook mailboxes for stealthy command-and-control. The Go-based malware authenticates via hardcoded Azure AD credentials to obtain OAuth2 tokens, then monitors a "Zomato Pizza" Outlook folder for emails with subject lines beginning with "Input." Encrypted commands are decrypted using AES-CBC and executed locally, with results returned via reply emails and original command messages deleted via HTTP DELETE requests. Symantec researchers found the Linux variant shares a near-identical codebase with the Windows version, including matching typos and the same AES key, confirming shared development [[BleepingComputer](https://www.bleepingcomputer.com/news/security/new-gogra-malware-for-linux-uses-microsoft-graph-api-for-comms/); [Cyberpress](https://cyberpress.org/castlerat-uses-deno-evasion/)]. Organizations should monitor Azure AD for suspicious OAuth token activity and unusual mailbox access patterns.

**[NEW] Phishing reemerges as top initial access vector in Q1 2026 IR engagements**
Cisco Talos Incident Response reports that phishing has returned as the most common initial access vector for the first time since Q2 2025, accounting for over a third of engagements where initial access could be determined—supplanting public-facing application exploitation, which dropped from 62% to 18% after ToolShell SharePoint vulnerabilities were patched. Notably, adversaries leveraged the Softr AI web application platform to generate credential harvesting pages targeting Microsoft Exchange and Outlook Web Access, marking the first documented use of a specific AI tool in a Talos IR incident. Public administration and healthcare tied as the most targeted industry verticals at 24% each, with public administration maintaining its top-target position for three consecutive quarters. Pre-ransomware incidents comprised only 18% of engagements, with no ransomware deployment due to early mitigation, though Rhysida actors deployed the uncommon MeowBackConn backdoor associated with Gootloader [[MalwareNews](https://malware.news/t/ir-trends-q1-2026-phishing-reemerges-as-top-initial-access-vector-as-attacks-targeting-public-administration-persist/106313)]. The report emphasizes MFA weaknesses (35% of engagements) and exposed management ports enabling rapid lateral movement.

**[NEW] DinDoor backdoor abuses Deno runtime and MSI installers to evade security**
A malware strain dubbed DinDoor, linked to the Iranian APT group Seedworm (MuddyWater) operating under the Tsundere Botnet umbrella, is using the legitimate Deno JavaScript runtime and malicious MSI installers to execute obfuscated scripts while bypassing traditional security controls that focus on PowerShell or Python execution. Samples analyzed by Hunt.io show two variants: one uses a double-extension PDF masquerade to drop a hidden PowerShell script that installs Deno locally, while a fileless version packaged with WiX toolsets displays a fake Windows error message while silently executing PowerShell in the background. Decoding a hardcoded JWT in the C2 URL path revealed campaign metadata pointing to `serialmenot[.]com`, a multi-tenant infrastructure shared among state-sponsored and criminal actors previously linked to CastleLoader malware. Researchers mapped 20 active malicious servers across bulletproof hosting providers using specific HTTP header patterns (Via: 1.1 Caddy) and 404 response anomalies [[Cyberpress](https://cyberpress.org/dindoor-evades-security-tools/)]. Organizations should block traffic to identified C2 infrastructure and monitor for MSI installer abuse.

## ⚠️ Vulnerabilities & Patches

**[NEW] Firefox 150 patches 41 security vulnerabilities including critical memory corruption**
Mozilla has released Firefox 150 with fixes for 41 security vulnerabilities, including critical use-after-free flaws in the DOM component (CVE-2026-6746) and WebRTC (CVE-2026-6747) that could lead to remote code execution. The update addresses multiple high-severity memory corruption bugs affecting browser stability and security, with Mozilla urging immediate updates for all users. The vulnerability details indicate attackers could exploit these flaws through malicious web content without requiring user interaction beyond browsing to a compromised site [[GBHackers](https://gbhackers.com/mozilla-firefox-150-released/)]. Organizations managing Firefox deployments should prioritize this update given the critical severity of the memory corruption issues.

**[NEW] Spring Authorization Server vulnerable to XSS and SSRF via dynamic client registration**
A critical vulnerability (CVE-2026-22752) in Spring Security Authorization Server allows attackers to inject malicious client metadata through Dynamic Client Registration endpoints, enabling stored XSS, privilege escalation, and Server-Side Request Forgery attacks. The flaw affects organizations running Dynamic Client Registration endpoints that accept client metadata without proper validation. Researcher Kelvin Mbogo responsibly disclosed the vulnerability, which has been addressed in patched versions. Organizations using Spring Authorization Server should review their configurations for exposed registration endpoints and apply the available security updates [[GBHackers](https://gbhackers.com/critical-spring-authorization-server-issue/)].

**[NEW] CrowdStrike LogScale self-hosted deployments vulnerable to unauthenticated file read**
A critical path traversal vulnerability (CVE-2026-40050, CVSS 9.8) in CrowdStrike LogScale allows remote attackers to read arbitrary files from affected servers without authentication, though the flaw only impacts self-hosted deployments and not the Next-Gen SIEM SaaS platform. CrowdStrike confirmed no evidence of active exploitation and has released patched versions 1.235.1, 1.234.1, 1.233.1, and LTS 1.228.2. The vulnerability stems from improper input validation in a specific cluster API endpoint and could allow attackers to extract configuration files, credentials, or other sensitive data [[Cyberpress](https://cyberpress.org/crowdstrike-logscale-vulnerability/)]. Organizations running self-hosted LogScale instances should upgrade immediately and ensure sensitive API endpoints are not internet-accessible.

**[NEW] 1,370+ SharePoint servers remain exposed to actively exploited spoofing vulnerability**
The Shadowserver Foundation has identified 1,370 internet-facing Microsoft SharePoint servers still vulnerable to CVE-2026-32201, a spoofing vulnerability added to CISA's Known Exploited Vulnerabilities catalog. This represents a decrease from 1,745 exposed systems on April 15, but hundreds of organizations remain at risk of network-level spoofing attacks that could enable unauthorized access to sensitive documents and lateral movement. The flaw stems from improper input validation that allows attackers to craft malicious requests processed as legitimate by the server [[Cyberpress](https://cyberpress.org/1370-microsoft-sharepoint-servers-exposed/)]. Organizations should apply Microsoft's security updates immediately and review SharePoint logs for suspicious activity patterns.

## 🛡️ Defense & Detection

**[NEW] Auraboros RAT framework discovered with exposed C2 panel supporting live audio and cookie theft**
Security researchers uncovered a previously undocumented remote access trojan framework called Auraboros with a fully exposed, unauthenticated command-and-control panel supporting live audio streaming, intensive keylogging, browser credential theft, and multi-cookie hijacking capabilities. The "Auraboros C2 Station" presents a polished single-page dashboard that immediately loads victim data and live controls without authentication, making all compromised endpoints and stolen data accessible to anyone who finds the panel. The discovery highlights how threat actors continue deploying infrastructure with minimal security hardening, creating opportunities for defenders and other attackers alike to access the same command infrastructure [[GBHackers](https://gbhackers.com/auraboros-rat-adds-live-audio/)]. Organizations should hunt for the disclosed C2 indicators and monitor outbound traffic to associated infrastructure.

**[NEW] Claude Desktop drops native messaging host manifest across Chromium browsers without consent**
Security researcher Alexander Hanff discovered that Anthropic's Claude Desktop application silently installs a Native Messaging host manifest into multiple Chromium browser profiles (Chrome, Edge, Brave, Arc, Vivaldi, Opera, Chromium) on macOS—including browsers not yet installed on the system. The manifest pre-authorizes three Chrome extension IDs to call a helper application with user-level privileges, enabling browser automation features including session access, DOM reading, and form filling without explicit consent. While Hanff characterizes this as "spyware," Malwarebytes notes it's more accurately described as expanding attack surface through a pre-positioned bridge that could potentially be abused. Anthropic has not published a technical privacy specification for this integration, and the behavior on Windows and Linux remains undocumented [[MalwareNews](https://malware.news/t/researcher-claims-claude-desktop-installs-spyware-on-macos/106315)]. Users should be aware of this integration and its potential security implications.

**[NEW] French fintech accounts weaponized as money laundering channels at scale**
Organized cybercriminal groups are abusing freelancer-focused French fintech platforms (Revolut, Wise, N26) to create verified mule accounts that move stolen funds across Europe within minutes using SEPA instant transfers. Group-IB research indicates nearly 1 in 7 business account sign-ups in France showed fraudulent patterns in confirmed customer data, with verified European fintech accounts sold on dark web markets for $300-700 each via escrow services. The scheme combines phishing for personal information with social engineering to open accounts that appear legitimate, enabling rapid fund movement before banks or victims detect fraud. Credit transfer fraud losses across the EEA reached €2.5 billion in 2024, a 24% increase from the previous year [[GBHackers](https://gbhackers.com/french-fintech-accounts/); [Cyberpress](https://cyberpress.org/french-fintech-funds-routed/)]. Financial institutions should enhance KYC verification for remote account opening and monitor for rapid sequential transactions characteristic of money laundering.

**[NEW] Global SIM Farm-as-a-Service operation spans 87 exposed panels across 17 countries**
Infrastructure intelligence firm Infrawatch has exposed a globally distributed SIM Farm-as-a-Service ecosystem powered by a Belarus-based software platform called ProxySmart, identifying 87 exposed control panels across 17 countries and at least 94 physical phone-farm locations. SIM farms—physical racks of smartphones or 4G/5G USB modems with active SIM cards—enable threat actors to bypass SMS-based verification, conduct social engineering at scale, and generate fraudulent accounts. The exposed panels allow registration, purchase of SIM access time, and configuration of services without authentication, providing turnkey infrastructure for account fraud, two-factor authentication bypass, and spam campaigns [[GBHackers](https://gbhackers.com/sim-farm-as-a-service-operation-spanning-87-panels-in-17-nations/)]. Organizations relying on SMS for authentication should consider the infrastructure compromised and implement alternative verification methods.
