# Daily Threat Intelligence Digest — May 16, 2026

## 🔴 Critical Threats & Active Exploitation

**[UPDATE] Cisco SD-WAN Zero-Day CVE-2026-20182 (CVSS 10.0) — Metasploit Module Released, CISA Deadline Tomorrow**

CVE-2026-20182, the seventh actively exploited Cisco SD-WAN vulnerability in 2026, now has a public Metasploit module released by Rapid7, dramatically lowering the barrier to widespread exploitation. The flaw — discovered March 9 but only disclosed May 14 — allows unauthenticated attackers to bypass peering authentication in the control connection handshake by claiming to be a vHub device. Success grants administrative access to NETCONF and the ability to manipulate routing and policy across the entire SD-WAN fabric. Rapid7's Douglas McKee described it as "the cybersecurity version of a Jedi mind trick." UAT-8616 remains the primary threat actor, but Cisco Talos has identified 10 additional threat clusters exploiting the broader CVE-2026-20122/20128/20133 chain since February. CISA's remediation deadline is May 17 — tomorrow. There are no workarounds. Cisco advises collecting admin-tech data before upgrading to preserve forensic evidence, auditing auth.log for unauthorized SSH keys, and escalating suspected compromises to Cisco TAC. [[CyberScoop](https://cyberscoop.com/cisco-sd-wan-zero-day-exploited/); [SOC Prime](https://socprime.com/blog/cve-2026-20182-analysis/); [Rapid7](https://www.rapid7.com/blog/post/2026/05/14/cve-2026-20182-cisco-sd-wan-authentication-bypass/)]

**[NEW] Microsoft Exchange Server Zero-Day CVE-2026-42897 — Active OWA Spoofing, No Permanent Patch**

Just 48 hours after Microsoft's first zero-day-free Patch Tuesday in nearly two years, a new actively exploited zero-day in on-premises Exchange Server was disclosed. CVE-2026-42897 (CVSS 8.1) is a cross-site scripting flaw in Outlook Web Access that allows an attacker to send a crafted email which executes arbitrary JavaScript in the victim's browser when opened under certain conditions. Exploitation has been confirmed in the wild — Microsoft has not disclosed the threat actor, targets, or campaign scale. Exchange Server 2016, 2019, and Subscription Edition are affected; Exchange Online is not. No permanent patch exists. Microsoft's emergency mitigation: enable the Exchange Emergency Mitigation Service (EEMS), which applies URL Rewrite protection automatically. For air-gapped environments, deploy the Exchange On-premises Mitigation Tool (EOMT). The speed with which this emerged after Patch Tuesday — and the attacker's choice to bypass server-side exploitation in favor of client-side JavaScript execution — suggests sophisticated targeting of OWA users. [[SecurityWeek](https://www.securityweek.com/microsoft-warns-of-exchange-server-zero-day-exploited-in-the-wild/); [SOC Prime](https://socprime.com/blog/cve-2026-42897-analysis/)]

**[NEW] Copy Fail CVE-2026-31431 — 9-Year-Old Linux Kernel LPE, CISA KEV Deadline Expired, PoC Public [r/cybersecurity, TLDR InfoSec, tl;dr sec]**

A local privilege escalation vulnerability present in the Linux kernel since 2017 — predating even Dirty Pipe — can grant any unprivileged local user root access in seconds. CVE-2026-31431 ("Copy Fail") exploits a logic flaw in the `algif_aead` AEAD crypto interface where a 2017 in-place optimization causes a four-byte write to land in the page cache of an arbitrary readable file, such as `/usr/bin/su`. PoC is a 700-byte Python script requiring only standard syscalls. Unlike Dirty Cow (race condition) and Dirty Pipe (write constraints), Copy Fail is deterministic and works across every major distribution. CISA added it to KEV with a May 15 deadline — which has already passed. Microsoft Defender has observed limited in-the-wild exploitation, primarily testing activity. Copy Fail is the third Dirty-Frag-class vulnerability disclosed in three weeks, following Dirty Frag (CVE-2026-43284/43500) and Fragnesia (CVE-2026-46300), all targeting different kernel subsystems with a shared page-cache corruption theme. Organizations that applied module blacklists for Fragnesia are not protected against Copy Fail — the only mitigation is a kernel patch. The attack leaves no disk footprint, defeating conventional file integrity monitoring. [[Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/05/01/cve-2026-31431-copy-fail-vulnerability-enables-linux-root-privilege-escalation/); [Tenable FAQ](https://www.tenable.com/blog/copy-fail-cve-2026-31431-frequently-asked-questions-about-linux-kernel-privilege-escalation); [Sysdig](https://www.sysdig.com/blog/cve-2026-31431-copy-fail-linux-kernel-flaw-lets-local-users-gain-root-in-seconds); r/cybersecurity (641↑)]

**[NEW] PraisonAI CVE-2026-44338 — Legacy API Auth Disabled by Default, Exploited Within Hours**

A high-severity authentication bypass in the PraisonAI AI agent framework was exploited within hours of its public disclosure, continuing the pattern of AI platforms shipping with dangerously insecure defaults. CVE-2026-44338 affects PraisonAI versions 2.5.6 through 4.6.33, where the legacy Flask-based API server hard-codes `AUTH_ENABLED=*** and binds to `0.0.0.0:8080` — a configuration the deployment templates actively recommend. Unauthenticated attackers can enumerate configured AI agents via `GET /agents` and execute arbitrary AI workflows via `POST /chat`, draining model quotas, accessing sensitive outputs, and potentially achieving downstream compromise through tool-calling integrations. Fixed in version 4.6.34. This is the third AI platform security incident this month following Langflow CVE-2026-33017 and Ollama Bleeding Llama (CVE-2026-7482), reinforcing that AI infrastructure defaults are systematically insecure. **[NEW] Funnel Builder WordPress Plugin Exploited to Steal Credit Cards — 40K+ Sites Affected**

A critical unauthenticated vulnerability in the Funnel Builder plugin for WooCommerce (40,000+ active installations) is being actively exploited to inject JavaScript payment card skimmers into checkout pages. Discovered by Sansec, the flaw allows attackers to modify the plugin's global settings through an unprotected checkout endpoint, injecting a skimmer disguised as Google Tag Manager (`analytics-reports[.]com/wss/jquery-lib.js`) that opens a WebSocket to `protect-wss[.]com/ws` to exfiltrate card numbers, CVVs, and billing addresses. **[NEW] Google Project Zero Details Pixel 10 Zero-Click Exploit Chain — Audio Decoding to Kernel Control**

Google Project Zero's Seth Jenkins and Jann Horn published a full zero-click exploit chain against Pixel 10 devices, progressing from a crafted Dolby Digital Plus audio stream (CVE-2025-54957, previously used against Pixel 9) to complete kernel control via a newly discovered vulnerability in the VPU driver for the Chips&Media Wave677DV block on the Tensor G5 chip. The VPU `mmap` handler performs zero bounds checking on the requested mapping size, allowing an attacker to map arbitrary physical memory including the entire kernel image. Because Pixel 10 places the kernel at a fixed physical address, no KASLR bruteforcing is required. Exploit was built in under a day. Google shipped a fix 71 days after reporting — the first time one of Jenkins' Android driver bugs was patched in under 90 days. However, the research underscores that shallow, easily discoverable vulnerabilities persist in vendor-maintained kernel drivers despite years of bug bounty programs. The Dolby UDC exploit works on devices that have not applied December 2025/January 2026 Android security patches. [[Cyber Security News](https://cyberpress.org/zero-click-exploit-chain-for-pixel-10/); [GBHackers](https://gbhackers.com/pixel-10-zero-click-exploit-chain/)]

## ⚠️ Vulnerabilities & Patches

**[NEW] Linux "ssh-keysign-pwn" CVE-2026-46333 — 6-Year Race Condition Exposes SSH Keys and /etc/shadow**

Qualys researchers have disclosed a 6-year-old race condition in the Linux kernel's `__ptrace_may_access()` function that allows unprivileged local attackers to steal SSH private keys and `/etc/shadow` password hashes. Tracked as CVE-2026-46333, the vulnerability exploits a window during process exit: a privileged process like `ssh-keysign` calls `exit_mm()` (nulling memory) before `exit_files()` (closing file descriptors). During this gap, an attacker uses `pidfd_getfd()` to hijack open FDs to sensitive files. Because dumpability evaluation incorrectly handles kernel threads and threads without active memory maps, no `CAP_SYS_PTRACE` capability is required. A public PoC is available on GitHub. Affects all Linux kernels before the May 14, 2026 commit merging the fix. Distributions including Ubuntu, Debian, Arch, CentOS, and Raspberry Pi OS are vulnerable. Until patching, restrict local user access and audit ptrace permissions. [[GBHackers](https://gbhackers.com/linux-ssh-keysign-pwn-flaw/)]

**[NEW] VMware Fusion CVE-2026-41702 (CVSS 7.8) — Local Root via TOCTOU Race Condition**

Broadcom has patched a local privilege escalation vulnerability in VMware Fusion version 25H2. CVE-2026-41702 is a Time-of-Check Time-of-Use race condition in a SETUID binary that allows low-privileged local attackers to execute code as root. Reported by researcher Mathieu Farrell (`@coiffeur0x90`), the flaw has low attack complexity and requires no user interaction. No workarounds exist — upgrade to VMware Fusion 26H1. While no active exploitation has been reported, local privilege escalation vulnerabilities in virtualization platforms are increasingly chained with initial access vectors. [[Cyber Security News](https://cyberpress.org/vmware-fusion-vulnerability/); [GBHackers](https://gbhackers.com/vmware-fusion-flaw-gain-root-privileges/)]

**[NEW] Android 16 VPN Bypass — Real IP Leak via QUIC Teardown Feature, Google Marks "Won't Fix"**

A critical privacy bypass in Android 16 allows any malicious app with only the auto-granted INTERNET and ACCESS_NETWORK_STATE permissions to leak a user's real IP address even with "Always-On VPN" and "Block connections without VPN" fully enabled. The attack exploits the new `registerQuicConnectionClosePayload` API in `ConnectivityService.java`, which has zero permission checks and bypasses VPN routing because `system_server` (UID 1000) is exempt from fwmark enforcement. Google closed the Android Vulnerability Reward Program report as "Won't Fix" on April 18. Users can mitigate via ADB: `device_config put tethering close_quic_connection -1` followed by a reboot. Journalists, activists, and privacy-conscious users are at highest risk. A PoC app is available on GitHub. [[Cyber Security News](https://cyberpress.org/android-16-vpn-bypass/)]

**[UPDATE] Microsoft Backpedals — Edge Will Stop Loading Cleartext Passwords Into Memory**

Microsoft has reversed its stance on Edge loading all saved passwords into process memory in cleartext at startup — previously defended as "by design" — and announced a defense-in-depth fix rolling to all Edge channels. The fix is already live in Canary and will ship with build 148 to Stable, Beta, Dev, and Extended Stable. Microsoft Edge Security Lead Gareth Evans cited the Secure Future Initiative and customer feedback as driving factors. The behavior, disclosed May 4 by researcher Tom Jøran Sønstebyseter Rønning, made Edge uniquely vulnerable among Chromium-based browsers to infostealers and process memory dumping. Organizations running Edge for enterprise browsing should accelerate adoption of the patched build.
