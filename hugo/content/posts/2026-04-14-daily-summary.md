# Daily Threat Intel Digest - April 14, 2026

## 🔴 Critical Threats & Active Exploitation

**[UPDATE] Adobe Acrobat Reader zero-day patched after active exploitation**
Adobe has released an out-of-band security update addressing a critical zero-day vulnerability in Acrobat Reader that attackers are actively exploiting in the wild. This follows yesterday's reporting on Adobe zero-day attacks, with today's patch confirming the threat has moved from limited exploitation to a broader risk requiring immediate action across enterprise environments. Organizations running Acrobat Reader should prioritize this update immediately given the confirmed active exploitation [[Adobe Patches Exploited Acrobat Reader Flaw](https://malware.news/t/adobe-patches-exploited-acrobat-reader-flaw/106022#post_1); [Organizations Warned of Exploited Windows, Adobe Acrobat Vulnerabilities](https://www.securityweek.com/organizations-warned-of-exploited-windows-adobe-acrobat-vulnerabilities/)].

**[NEW] CISA confirms active exploitation of Fortinet SQL injection flaw**
CISA has added CVE-2026-21643, a critical SQL injection vulnerability in Fortinet software, to its Known Exploited Vulnerabilities catalog. The agency's April 13, 2026 confirmation indicates threat actors are actively exploiting this weakness in real-world attacks, requiring federal agencies and critical infrastructure operators to apply patches immediately under binding operational directive timelines. Organizations running affected Fortinet appliances should treat this as an emergency patching priority given the KEV catalog designation confirms in-the-wild exploitation [[CISA Warns Fortinet SQL Injection Flaw Is Being Actively Exploited](https://gbhackers.com/cisa-warns-fortinet-sql-injection-flaw/)].

**[NEW] Critical Apache Tomcat Tribes regression enables unauthenticated RCE**
CVE-2026-34486 introduces a dangerous "fail-open" regression in Apache Tomcat Tribes clustering that can turn an exposed cluster receiver into an unauthenticated remote code execution opportunity. The flaw was introduced by a prior security fix for CVE-2026-29146, where a one-line control-flow change caused decryption failures to forward attacker-controlled bytes to Java deserialization code instead of dropping them. Affected versions include Tomcat 11.0.20 (fixed in 11.0.21), 10.1.53 (fixed in 10.1.54), and 9.0.116 (fixed in 9.0.117). Organizations running Tomcat Tribes with EncryptInterceptor should immediately restrict TCP port 4000 to trusted cluster members and patch to the fixed releases, as the attack chain requires only network reachability and usable deserialization gadgets on the classpath [[CVE-2026-34486: Apache Tomcat Tribes Regression Creates Unauthenticated RCE Path](https://malware.news/t/cve-2026-34486-apache-tomcat-tribes-regression-creates-unauthenticated-rce-path/106025#post_1)].

**[NEW] Rockstar Games breach exposes 78.6M records via supply chain compromise**
ShinyHunters exploited a third-party integration flaw in Anodot, an AI-powered cloud cost monitoring platform, to access Rockstar Games' connected Snowflake data warehouse and exfiltrate 78.6 million internal records. The attackers extracted authentication tokens from Anodot's environment to impersonate legitimate services and gain silent access to Rockstar's data infrastructure. The leaked dataset contains non-sensitive analytics data for GTA Online and Red Dead Online, including game metrics and revenue segmentation, but no passwords, payment information, personal data, or GTA 6 development assets. This incident underscores the persistent danger of SaaS integration attacks where API keys and authentication tokens enable indirect access to cloud infrastructure [[Rockstar's GTA Data Breach Exposes 78.6 Million Records Online](https://cyberpress.org/rockstars-gta-data-breach-exposes-78-6-million-records-online/); [Rockstar's GTA Game Hacked, 78.6 Million Records Published Online](https://gbhackers.com/rockstars-gta-game-hacked/)].

## 🎯 Threat Actor Activity & Campaigns

**[NEW] APT41 deploys stealthy Linux backdoor targeting cloud credentials across major providers**
A previously undocumented Linux ELF backdoor attributed to China-linked threat group APT41 (Winnti) is actively targeting cloud workloads across AWS, GCP, Azure, and Alibaba Cloud with zero detections on security platforms at time of discovery. The stripped, statically-linked 64-bit executable harvests credentials by querying internal metadata services and scanning local configuration files, then exfiltrates encrypted data over SMTP port 25 to blend with normal network traffic. The backdoor uses a selective handshake mechanism where the C2 server only responds to infected machines presenting valid authentication tokens, making malicious infrastructure invisible to internet scanners. The campaign demonstrates APT41's six-year evolution from simple Linux backdoors to advanced cloud-aware threats [[APT41 Turns Linux Cloud Servers Into Credential Theft Targets With New Winnti Backdoor](https://cyberpress.org/apt41-targets-linux-clouds/); [APT41 Targets Linux Cloud Servers With New Winnti Backdoor](https://gbhackers.com/new-winnti-backdoor/)].

**[NEW] VIPERTUNNEL Python backdoor deployed via obfuscated loader chain linked to EvilCorp**
Investigators discovered a sophisticated Python-based persistence mechanism during a DragonForce ransomware intrusion, using a malicious sitecustomize.py file that executes a fake DLL whenever pythonw.exe starts. The obfuscated multi-stage loader decrypts and deploys VIPERTUNNEL, a Python SOCKS5 proxy backdoor that establishes an outbound tunnel over TCP port 443 with hardcoded C2 credentials. The campaign is tightly linked to UNC2165/EvilCorp and former RansomHub affiliates, demonstrating how Python-centric obfuscation stacks and living-off-the-land execution provide long-term access for ransomware operations. Security teams should hunt for anomalous Python autoruns, unexpected sitecustomize.py usage, and Base85-plus-crypto loader patterns [[Obfuscated Loader Chain Delivers VIPERTUNNEL Backdoor via Fake DLL](https://cyberpress.org/vipertunnel-backdoor-via-fakedll/)].

**[NEW] Fake Proxifier GitHub installer delivers ClipBanker cryptocurrency malware**
Attackers are distributing a trojanized Proxifier installer through a GitHub repository that appears legitimate, hiding malware in release files alongside fake activation keys. The multi-stage infection chain weakens Microsoft Defender by injecting code to add antivirus exclusions, then launches the real Proxifier installer to avoid suspicion while executing obfuscated PowerShell scripts stored in the registry. The final payload is ClipBanker, a C++ clipboard hijacker that monitors for cryptocurrency wallet addresses and replaces them with attacker-controlled ones across Bitcoin, Ethereum, Monero, Solana, and TRON networks. Researchers identified over 2,000 detections among Kaspersky users since early 2025, primarily in India and Vietnam [[Hackers Distribute ClipBanker Malware via Fake Proxifier Installer On GitHub](https://cyberpress.org/clipbanker-spread-via-proxifier/); [Fake Proxifier GitHub Installer Spreads ClipBanker Crypto Malware](https://gbhackers.com/fake-proxifier-github/)].

**[NEW] Okta identity systems targeted via voice-based social engineering**
Hackers are shifting away from traditional email phishing to directly target Okta and other identity providers using "Okta vishing"—voice-based social engineering that tricks users into revealing credentials or approving MFA prompts over phone calls. This approach transforms what would be a single account compromise into an immediate organization-wide cloud data breach via Single Sign-On access, as attackers gain control over the central identity layer that governs access to all connected applications. Security teams should implement additional verification steps for identity provider changes and train staff to recognize social engineering attempts targeting authentication flows [[Okta Under Attack as Hackers Skip Phishing for Identity Systems](https://gbhackers.com/okta-under-attack/)].

## ⚠️ Vulnerabilities & Patches

**[NEW] Critical Synology SSL VPN Client flaws enable credential theft**
Synology has patched two high-impact vulnerabilities in its SSL VPN Client utility under advisory Synology-SA-26:05. CVE-2021-47960 (CVSS 6.5) allows attackers to access sensitive system files by exploiting improper access permissions on the client's installation directory, while CVE-2021-47961 (CVSS 8.1) stores user passwords in plaintext within local configuration files, enabling credential extraction. Both flaws require user interaction through social engineering, such as tricking victims into visiting malicious websites that exploit a local HTTP server on the loopback interface. Users must upgrade to version 1.4.5-0684 or later, as no workarounds exist [[Critical Synology SSL VPN Client Flaw Lets Remote Attackers Access Sensitive Files](https://cyberpress.org/synology-ssl-vpn-client-flaw/); [Synology SSL VPN Client Vulnerability Enabled Remote Access to Sensitive Files](https://gbhackers.com/synology-ssl-vpn-client-vulnerability/)].
