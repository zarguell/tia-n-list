# Daily Threat Intel Digest - April 29, 2026

## 🔴 Critical Threats & Active Exploitation

**[NEW] VECT 2.0 ransomware permanently destroys enterprise files over 128 KB due to cryptographic flaw**
A critical flaw in the VECT 2.0 Ransomware-as-a-Service (RaaS) operation transforms the malware into a data wiper, permanently destroying files larger than 128 KB and rendering ransom payment useless for recovery. The malware splits files into four chunks and generates a unique ChaCha20-IETF nonce for each, but a memory buffer overwrite discards the first three nonces, making mathematical decryption impossible for both victims and attackers. The cross-platform malware targets the full enterprise stack—Windows, Linux, and VMware ESXi—manipulating Safe Mode boot settings, wiping system logs, and aggressively shutting down hypervisor monitoring services. VECT has established formal partnerships with BreachForums and TeamPCP (the same group behind recent supply chain attacks on Trivy and Checkmarx KICS), providing a ready-made pipeline to exploit thousands of downstream consumers. Security teams must shift response strategy from ransom negotiation to strict backup isolation and endpoint monitoring, as data recovery is fundamentally impossible. [[Cyberpress](https://cyberpress.org/vect-2-0-multi-platform-threat/); [GBHackers](https://gbhackers.com/vect-2-0-ransomware-wipes/)]

**[UPDATE] CISA adds actively exploited Windows Shell zero-day (CVE-2026-32202) to KEV catalog**
CISA added CVE-2026-32202, a zero-click Windows Shell vulnerability, to its Known Exploited Vulnerabilities catalog on April 28, ordering federal agencies to patch by May 12. Akamai research reveals the flaw is an incomplete patch remnant from February's CVE-2026-21510 RCE fix, leaving an authentication coercion vulnerability that enables credential theft via auto-parsed LNK files. Russian APT28 (Fancy Bear) previously exploited the initial RCE flaw in December 2025 attacks against Ukraine and EU countries as part of an exploit chain also targeting an LNK file vulnerability (CVE-2026-21513). The vulnerability enables network spoofing, credential harvesting, and lateral movement across compromised networks, putting any organization running unpatched Windows systems at risk of stealthy intrusion. Separately, three recently leaked Windows zero-days (BlueHammer, RedSun, and UnDefend) remain under active exploitation for privilege escalation, with the latter two still lacking patches. [[BleepingComputer](https://www.bleepingcomputer.com/news/security/cisa-orders-feds-to-patch-windows-flaw-exploited-in-zero-day-attacks/); [Cyberpress](https://cyberpress.org/cisa-alerts-on-microsoft-windows/); [GBHackers](https://gbhackers.com/cisa-warns-of-windows-shell-zero-day/)]

**[NEW] Vimeo confirms data breach via third-party analytics vendor Anodot, linked to ShinyHunters**
Vimeo confirmed a data breach stemming from a compromise of its third-party analytics provider Anodot, exposing user email addresses, video titles, metadata, and technical data—though passwords, financial data, and video content remained secure. Google Threat Intelligence has linked the incident to the ShinyHunters threat group, which systematically targets SaaS platforms through third-party compromises to maximize downstream impact. Attackers infiltrated Anodot's centralized analytics systems to gain indirect access to multiple organizations simultaneously, with stolen data typically weaponized for extortion or sold on underground marketplaces. Vimeo has revoked all Anodot-related credentials and removed the integration entirely. While no authentication data was exposed, the breach elevates phishing risk for affected users and underscores the persistent supply chain vulnerability inherent in deeply embedded third-party integrations. [[Cyberpress](https://cyberpress.org/vimeo-confirms-data-breach/); [GBHackers](https://gbhackers.com/vimeo-confirms-data-breach/)]

**[NEW] Critical GitHub RCE vulnerability (CVE-2026-3854) exposed millions of repositories**
Wiz Research identified a critical remote code execution vulnerability in GitHub's internal git infrastructure that allowed any authenticated user to execute arbitrary commands on backend servers via a single standard git push operation. The flaw stems from improper neutralization of special elements—specifically, push option parameters containing an unsanitized semicolon (;) delimiter—enabling attackers to inject commands into internal metadata processing flows. The vulnerability affected both GitHub.com and GitHub Enterprise Server (GHES) on-premise installations, putting repository infrastructure, source code, and CI/CD pipelines at risk of full compromise. [[GBHackers](https://gbhackers.com/github-com-and-enterprise-server-vulnerability/); [CERT-AGID](https://cert-agid.gov.it/news/github-e-github-enterprise-server-vulnerabilita-rce-cve-2026-3854/)]

## 🎯 Threat Actor Activity & Campaigns

**[NEW] BlueNoroff targets cryptocurrency executives with AI-generated deepfakes and fileless PowerShell**
North Korea's BlueNoroff (Lazarus subgroup) is conducting a global campaign against Web3 and cryptocurrency organizations, using AI-generated deepfake video calls and fileless PowerShell attacks to compromise high-value targets. The attack chain begins with spear-phishing via compromised Telegram accounts impersonating previous victims, delivering typo-squatted Zoom/Teams links through Calendly invitations. During fake meetings, victims encounter a fabricated SDK error prompting them to copy a "diagnostic command" that actually executes a fileless PowerShell script, establishing persistent memory-resident C2 access. Post-exploitation modules steal Telegram Desktop sessions for lateral social engineering, inject AES-encrypted shellcode into Chromium browsers to extract cryptocurrency wallet extensions and credentials, and exfiltrate data via customized Telegram Bot API. Arctic Wolf identified over 100 targeted individuals across 20 countries, with nearly half holding CEO or founder titles. [[Cyberpress](https://cyberpress.org/bluenoroff-fileless-powershell-campaign/); [GBHackers](https://gbhackers.com/ai-generated-zoom-lure/)]

**[UPDATE] Sandworm deploys double-encrypted SSH-over-Tor tunneling for persistent network access**
Russia's Sandworm (APT-C-13) has advanced its persistence capabilities with a double-encrypted tunnel system using SSH and Tor to create invisible "shadow management consoles" on victim networks, directly connecting internal ports (445 for SMB, 3389 for RDP) to external attacker infrastructure without visible firewall modifications. The attack chain begins with malicious LNK files disguised as official documents in spear-phishing ZIP archives, executing a master control script that performs sandbox detection before deploying the tunneling infrastructure. The malware uses the obfs4 protocol to disguise Tor traffic as normal web browsing, bypassing deep packet inspection and network monitoring. Scheduled tasks ensure automatic startup on reboot, while pre-loaded SSH keys grant passwordless access for continuous espionage and lateral movement. [[Cyberpress](https://cyberpress.org/sandworm-enables-hidden-access/)]

**[NEW] SLOTAGENT RAT deploys advanced anti-forensics and BOF execution capabilities**
A newly identified Remote Access Trojan dubbed SLOTAGENT demonstrates sophisticated evasion and post-exploitation capabilities, including Beacon Object File (BOF) execution popularized by Cobalt Strike, allowing attackers to expand their network footprint without writing new executables to disk. The malware employs custom API hashing using XOR and ROR11 operations to dynamically resolve Windows API functions, while encrypting nearly all internal text strings with a Tiny Encryption Algorithm variant that can only be decrypted in memory during active execution. SLOTAGENT communicates with C2 infrastructure at 43.156.59.110:699 using custom JSON-formatted protocols, transmitting comprehensive host profiling including hardware MAC addresses, privilege levels, and process IDs. Researchers have published a custom IDAPython decryption script to enable static analysis without payload detonation. [[Cyberpress](https://cyberpress.org/slotagent-challenges-malware-analysis/); [GBHackers](https://gbhackers.com/slotagent-malware-hides-api/)]

**[NEW] LofyStealer targets Minecraft players with Node.js loader and memory injection**
The Brazilian LofyGang cybercrime group has returned with an upgraded Malware-as-a-Service infostealer operation targeting young Minecraft players through fake hacking tools disguised with official game icons. The two-stage architecture uses a 53.5 MB Node.js runtime bundle to evade sandbox analysis, decrypting and injecting a 1.4 MB native C++ payload directly into memory using direct syscalls to bypass EDR hooks. The payload targets eight Chromium-based browsers to extract cookies, saved passwords, authentication tokens, and financial data, exfiltrating via hidden PowerShell commands to a Brazilian-hosted C2 server operating on port 8080 with a branded web panel offering both free and premium tiers. [[Cyberpress](https://cyberpress.org/lofystealer-targets-minecraft-gamers/); [GBHackers](https://gbhackers.com/lofystealer-targets-minecraft-players/)]

## ⚠️ Vulnerabilities & Patches

**[NEW] Google Chrome 147 patches 30 vulnerabilities including multiple critical use-after-free RCE flaws**
Google has released Chrome 147.0.7727.137 addressing 30 security vulnerabilities, including critical use-after-free flaws in Canvas (CVE-2026-7363), iOS builds (CVE-2026-7361), Accessibility (CVE-2026-7344), and Views (CVE-2026-7343) that enable remote code execution through crafted web pages. High-severity patches include a $16,000-rewarded GPU use-after-free (CVE-2026-7333), heap buffer overflow in Skia (CVE-2026-7353), out-of-bounds read/write in ANGLE (CVE-2026-7354), and type confusion in the V8 JavaScript engine (CVE-2026-7337). These vulnerabilities are particularly dangerous when chained—attackers could trigger memory corruption via the Canvas component and achieve arbitrary code execution through V8 type confusion in browser-based drive-by attacks. [[Cyberpress](https://cyberpress.org/critical-google-chrome-flaws-allow-remote-code-execution-exploits/)]

**[NEW] cPanel releases emergency patch for critical authentication bypass vulnerability**
cPanel has issued an emergency security update for a critical authentication flaw affecting multiple authentication paths within the cPanel and WebHost Manager (WHM) ecosystem. Given that cPanel serves as the central control panel for web hosting environments, successful exploitation could grant attackers control over hosted websites, databases, email accounts, and server configurations. Web hosting administrators should apply the patch immediately. [[GBHackers](https://gbhackers.com/cpanel-releases-emergency-patch/)]

## 🛡️ Defense & Detection

**[NEW] Mozilla and Anthropic collaborate to identify 271 zero-days in Firefox using Claude Mythos AI**
Mozilla's collaboration with Anthropic has yielded extraordinary results, with the Claude Mythos AI model identifying and helping fix 271 security vulnerabilities in Firefox 150, following an earlier scan with Opus 4.6 that found 22 security-sensitive bugs in Firefox 148. The findings demonstrate that advanced AI models can systematically identify latent vulnerabilities in hardened targets at unprecedented scale, fundamentally shifting the economics of offensive security. Mozilla's experience suggests that while the volume of findings initially creates "vertigo" for security teams, organizations that reprioritize around AI-assisted vulnerability discovery can gain a decisive defensive advantage. [[Schneier on Security](https://www.schneier.com/blog/archives/2026/04/claude-mythos-has-found-271-zero-days-in-firefox.html)]

**[NEW] AI-powered honeypots exploit attacker automation to deceive malicious AI agents**
Cisco Talos research demonstrates how generative AI can rapidly deploy convincing, adaptive honeypots that specifically target AI-driven attacks, which prioritize speed over stealth and lack environmental awareness. Defenders can use simple text prompts to create diverse deceptive environments—from Linux shells to IoT devices—that trick automated threat agents into revealing their methodologies. The approach shifts defensive strategy from passive detection to active manipulation, creating a "hall of mirrors" where attackers unknowingly interact with simulated systems. The limiting factor is no longer tooling but the credibility of the simulated environment, though the technique specifically targets AI agents rather than skilled human operators who would detect inconsistencies. [[Talos Intelligence](https://blog.talosintelligence.com/ai-powered-honeypots-turning-the-tables-on-malicious-ai-agents/)]
