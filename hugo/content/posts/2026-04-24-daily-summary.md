# Daily Threat Intel Digest - April 24, 2026

## 🔴 Critical Threats & Active Exploitation

**[NEW] Bitwarden CLI compromised in supply chain attack via GitHub Actions**
Attackers exploited a GitHub Actions workflow in Bitwarden’s CI/CD pipeline to inject a credential harvester into the official `@bitwarden/cli` npm package (version 2026.4.0), stealing GitHub tokens, AWS/Azure/GCP cloud credentials, SSH private keys, and npm configuration files from developer systems. The malware uploads stolen data to attacker-controlled repositories on compromised GitHub accounts, using a Russian locale check as a kill-switch to avoid execution on systems with Russian environment settings [[Bitwarden CLI Hit by Supply Chain Attack Through GitHub Actions](https://cyberpress.org/bitwarden-cli-hit/); [Bitwarden NPM Package Hit in Supply Chain Attack](https://www.securityweek.com/bitwarden-npm-package-hit-in-supply-chain-attack/); [Bitwarden CLI Compromised After Malicious GitHub Actions Workflow](https://gbhackers.com/bitwarden-cli-compromised/)]. The incident is part of the broader Checkmarx-related supply chain campaign and highlights the risk of over-privileged CI/CD pipelines—organizations must immediately remove the compromised package, rotate all potentially exposed secrets, and audit GitHub accounts for unauthorized repositories with naming conventions like "fremen," "sandworm," or "mentat."

**[UPDATE] React2Shell exploitation scales to 900+ confirmed compromises with AI-assisted automation**
The mass exploitation campaign targeting CVE-2025-55182 has expanded significantly, with an exposed Bissa scanner server revealing over 900 confirmed compromises and 13,000 files detailing structured credential harvesting operations [[Hackers Track 900+ React2Shell Exploits via Telegram Bots](https://gbhackers.com/900-react2shell-exploited/); [Cybercriminals Track React2Shell Successes Using Telegram](https://cyberpress.org/telegram-tracks-exploit-successes/)]. Attackers are using Claude Code and OpenClaw AI tools to debug exploit code and triage stolen credentials, while Telegram bots send real-time alerts to a single operator tracking victim identities, security posture, and stolen data. The operation has compromised payroll records, HR data, and financial transactions from AI platforms, cloud providers, and payment systems. This escalation from opportunistic scanning to organized mass credential theft underscores the need to move credentials out of environment files into secret managers, patch React Server Components immediately, and control outbound traffic to prevent data exfiltration to attacker infrastructure.

**[NEW] Critical Python asyncio vulnerability enables arbitrary code execution on Windows**
A high-severity out-of-bounds write flaw in Python’s `sock_recvfrom_into()` method affects all Windows systems using the default `ProactorEventLoop` (Python 3.8+), enabling attackers to overwrite adjacent memory regions and potentially achieve arbitrary code execution through specially crafted network responses exceeding allocated buffer sizes [[Critical Python Vulnerability Enables Out-of-Bounds Write on Windows Systems](https://cyberpress.org/critical-python-vulnerability/); [Python Vulnerability Enables Out-of-Bounds Write on Windows](https://gbhackers.com/python-vulnerability-enables-out-of-bounds/)]. The vulnerability (CVE-2026-3298) specifically impacts Windows-hosted Python web servers, API backends, and asynchronous applications using UDP socket communication—Linux and macOS systems are unaffected. A patch is available via CPython PR #148809, and developers should avoid using the vulnerable function with the `nbytes` parameter in exposed environments until upgrades are complete.

**[NEW] Trigona ransomware affiliates deploy custom stealth exfiltration tool**
Trigona ransomware operators have abandoned common exfiltration utilities like Rclone and MegaSync in favor of a proprietary data-stealing tool designed for precision, speed, and evasion, allowing attackers to selectively target high-value folders containing invoices and sensitive PDFs while evading signature-based detection [[Hackers Deploy New Exfiltration Tool In Ransomware Attacks](https://cyberpress.org/ransomware-hackers-steal-data/); [Ransomware Gang Unveils Custom Data-Theft Tool](https://gbhackers.com/custom-data-theft-tool/)]. The custom uploader uses parallel streams (5 concurrent connections), automatic connection rotation after 2GB transfers, granular file filtering to exclude bulky media files, and integrated authentication to secure stolen data repositories. Attackers also deploy BYOVD (Bring Your Own Vulnerable Driver) techniques using Huorong Network Security Suite tools, PCHunter, and YDark to terminate endpoint protection at the kernel level. This shift toward custom tooling indicates increasing technical maturity among RaaS affiliates and complicates detection efforts.

**[NEW] Void Dokkaebi spreads malware through self-propagating coding test repositories**
North Korean threat actor Void Dokkaebi (Famous Chollima) is running a worm-like campaign that turns fake job interviews into self-spreading malware, where compromised developers unknowingly seed malicious `.vscode/tasks.json` files into their public repositories, creating new infection points for subsequent victims who clone the code [[Void Dokkaebi Targets Job Seekers With Malware Hidden In Coding Challenges](https://cyberpress.org/malware-in-coding-challenges/); [Void Dokkaebi Hackers Spread Malware Through Fake Job Interviews](https://gbhackers.com/void-dokkaebi-hackers/)]. The group targets developers with access to cryptocurrency wallets, signing keys, and CI/CD infrastructure. Researchers identified 750+ repositories carrying obfuscated JavaScript loaders and 392 malicious `tasks.json` files across GitHub, GitLab, and Bitbucket. Real-world victims included DataStax and Neutralinojs, where the attack went undetected for three days. The payload (DEV#POPPER RAT) communicates over WebSocket and deliberately avoids CI/CD environments to evade pipeline scans.

**[NEW] Ollama AI model upload flaw leaks server data without authentication**
An unauthenticated attacker can exploit a critical vulnerability in Ollama’s model quantization engine (CVE-2026-5757) to steal sensitive server data by uploading a maliciously crafted AI model file, affecting organizations running the popular open-source platform for local LLM deployment [[Hackers Exploit Ollama Model Uploads to Leak Server Data](https://gbhackers.com/hackers-exploit-ollama-model-uploads-to-leak-server-data/)]. The flaw requires no authentication and can be exploited through the model upload functionality, enabling data theft from servers hosting AI inference workloads. Organizations using Ollama should immediately review model upload policies, restrict upload capabilities to authenticated users, and monitor for unauthorized model deployments.

**[NEW] UNC6692 impersonates IT helpdesk on Microsoft Teams to deploy SNOW malware suite**
A newly identified threat group is combining email bombing campaigns with Microsoft Teams impersonation, flooding targets with spam emails before contacting victims via Teams as fake IT support offering "local patches" to resolve the issue [[Hackers Exploit Microsoft Teams to Breach Organizations While Posing as IT Helpdesk Staff](https://cyberpress.org/hackers-exploit-microsoft-microsoft-teams/); [Hackers Impersonate IT Helpdesk Staff to Breach Firms via Microsoft Teams](https://gbhackers.com/hackers-impersonate-it-helpdesk-staff-microsoft-teams/)]. The attack deploys a modular toolkit called the "SNOW" ecosystem: SNOWBELT (persistent Chromium extension), SNOWGLAZE (Python-based encrypted tunneling), and SNOWBASIN (RAT with screenshot, command execution, and exfiltration capabilities). After lateral movement via Python scripts and LSASS memory dumping, attackers use legitimate forensic tools like FTK Imager to exfiltrate the Active Directory database via LimeWire. Organizations should restrict external Teams communications, strengthen browser monitoring, and implement threat detection beyond standard antivirus solutions.

## 🎯 Threat Actor Activity & Campaigns

**[NEW] GLOBAL GROUP ransomware artifacts reveal ESXi-focused hypervisor attack playbook**
Analysis of leaked BlackLock/GLOBAL GROUP infrastructure reveals active preparation for ESXi hypervisor attacks using a FAKE_CAPTCHA LNK dropper paired with a Fragtor loader, designed to achieve "recovery denial" by encrypting entire virtualized estates in minutes [[Analyzing GLOBAL GROUP (BlackLock) Artifacts](https://malware.news/t/analyzing-global-group-blacklock-artifacts/106397#post_1)]. The domains `paksecurity[.]org` and `techoption[.]org` host payload infrastructure for campaigns targeting Windows, Linux, and ESXi encryptors with ChaCha20 encryption and RSA-OAEP key wrapping. Hypervisor attacks represent 43% of ransomware incidents in 2025, up from 29% the prior year, as operators target the "Tier-0" layer where EDR visibility is limited. Organizations must treat ESXi hosts as Tier-0 assets, avoid joining them directly to Active Directory, enable lockdown mode, deploy immutable backups, and monitor for anomalous `esxcli` commands or "ESX Admins" group creation.

**[NEW] Citizen Lab exposes global SS7/Diameter surveillance campaigns by commercial vendors**
A multi-year investigation reveals how commercial surveillance vendors are weaponizing SS7 and Diameter signaling protocol weaknesses to track individuals worldwide without device access, exploiting the trust-based architecture of mobile networks to send "Provide Subscriber Information" queries revealing exact cell tower locations [[Hackers Exploit SS7 and Diameter Protocols to Track Mobile Users Worldwide](https://cyberpress.org/hackers-exploit-ss7-and-diameter-protocols-to-track-mobile-users-worldwide/); [Hackers Exploit SS7 and Diameter Flaws to Track Mobile Users Globally](https://gbhackers.com/hackers-exploit-ss7-and-diameter-flaws/)]. Campaigns identified as STA1 and STA2 used spoofed operator identities across nine countries to target telecom executives, with attackers impersonating legitimate providers as "Ghost Operators" to blend malicious traffic with normal roaming activity. The attacks enable real-time location tracking, call interception, and two-factor authentication bypass without malware deployment. Telecom operators must deploy unified signaling firewalls capable of analyzing cross-protocol traffic to detect anomalies and block unauthorized queries.

**[NEW] China-linked hackers build covert networks through compromised SOHO routers**
China-nexus threat actors are increasingly leveraging large-scale botnets of hacked SOHO routers, IoT devices, and edge infrastructure to build covert proxy networks for stealthy cyber operations, avoiding detection by routing traffic through devices that appear as ordinary residential connections [[China-Linked Hackers Hide Behind Compromised Routers](https://gbhackers.com/compromised-routers/)]. This tactic complicates attribution and bypasses IP reputation checks, as traffic originates from thousands of compromised home and small business devices rather than data center infrastructure. The approach enables sustained covert operations while masking true attack origins behind layers of residential proxy infrastructure.

## ⚠️ Vulnerabilities & Patches

**[NEW] PhantomRPC technique enables privilege escalation across all Windows versions**
A new Windows RPC vulnerability enables local privilege escalation from Network Service or Local Service accounts to SYSTEM by deploying malicious RPC servers that mimic legitimate service endpoints, exploiting an architectural weakness where the RPC runtime does not verify server legitimacy [[PhantomRPC: A new privilege escalation technique in Windows RPC](https://malware.news/t/phantomrpc-a-new-privilege-escalation-technique-in-windows-rpc/106396#post_1)]. Five exploitation paths demonstrate escalation via Group Policy Service coercion, Edge browser startup, WDI background services, ipconfig execution, and Windows Time service interactions. Microsoft assessed the issue as moderate severity and declined to issue a CVE, noting the requirement for existing SeImpersonatePrivilege access. Organizations can mitigate by monitoring RPC exceptions via ETW, reducing unnecessary SeImpersonatePrivilege grants, and ensuring legitimate RPC servers are running where expected.

## 🛡️ Defense & Detection

**[NEW] Mythos-ready briefing elevates credential security as critical AI-era control**
A coalition of 60+ CISOs and security leaders from Google, Snowflake, Atlassian, and government agencies released a briefing responding to Anthropic's Mythos AI vulnerability discovery, naming secrets rotation, non-human identity governance, and honeytoken-based detection as critical controls for the AI vulnerability era [[What the Mythos-Ready Briefing Says About Credentials](https://blog.gitguardian.com/what-the-mythos-ready-briefing-says-about-credentials/)]. The briefing notes that credentials remain the leading initial access vector (22% of breaches) despite zero-day concerns, and that 29 million new hardcoded secrets were exposed on GitHub in 2025—with AI service credentials surging 81%. Organizations must inventory non-human identities (outnumbering human users 25-50x), deploy continuous secrets detection across repos and CI/CD systems, and implement phishing-resistant MFA for privileged accounts to limit blast radius when exploitation occurs.

**[NEW] Claude Desktop silently installs browser automation hooks without user consent**
Anthropic’s Claude Desktop application for macOS silently installs a Native Messaging bridge across Chromium-based browsers, establishing out-of-sandbox browser automation hooks that pose privacy and security risks by bypassing explicit user consent and standard application security boundaries [[Claude Desktop Reportedly Adds Browser Access Bridge for Chromium Browsers](https://gbhackers.com/claude-desktop-reportedly-adds-browser-access/)]. The unprompted installation creates potential for unauthorized browser control and data access. Organizations managing macOS devices should review Claude Desktop deployments and assess whether the browser integration aligns with security policies.

**[NEW] Fake CAPTCHA scam drives international SMS revenue fraud**
Attackers are abusing fake CAPTCHA pages to trick users into sending premium-rate international SMS messages, generating revenue through international revenue share fraud (IRSF) by triggering prefilled messages to multiple international numbers during "prove you're human" workflows [[Fake CAPTCHA Scam Triggers Costly SMS Fraud](https://gbhackers.com/fake-captcha-scam/); [Cybercriminals Exploit CAPTCHA Pages To Drive SMS Charges](https://cyberpress.org/captcha-pages-drive-sms/)]. The scam has been active since June 2020, with one test case producing 60 SMS messages costing approximately $30 in charges appearing on phone bills weeks later. The campaign uses traffic distribution systems (TDS) to target users and back-button hijacking to trap victims in redirect loops. Google announced a new spam policy targeting back-button hijacking with enforcement beginning June 15, 2026.

**[NEW] Credential stuffing tools industrialize account takeover through residential proxies**
Criminals are industrializing credential stuffing attacks using OpenBullet and SilverBullet frameworks loaded with combolists from infostealer logs, routing traffic through residential proxies to bypass rate limiting and IP reputation checks [[Credential stuffing tools and how to stop them](https://malware.news/t/credential-stuffing-tools-and-how-to-stop-them/106395#post_1)]. Residential proxies make attack traffic appear to come from ordinary home connections, evading bot detection systems. Defense requires anti-malware tools to stop infostealer infections at the endpoint, unique passwords with MFA, and organizational controls combining device intelligence, bot detection, and phishing-resistant authentication.

**[NEW] Telegram session hijacking via Pastebin PowerShell script reveals attacker testing**

The lack of obfuscation and private LAN addresses in related web collectors suggest this is operator validation rather than production deployment, but demonstrates the attack path for session-based account takeover.

**[NEW] Auraboros RAT panel exposed with unauthenticated live audio streaming**
