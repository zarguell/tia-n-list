## 🔴 Critical Threats & Active Exploitation

**[NEW] "Dirty Frag" — another universal Linux root zero-day, same class as Copy Fail, PoC public with no patches available**

A new Linux kernel privilege escalation vulnerability dubbed "Dirty Frag" allows local attackers to gain root on all major distributions with a single command. Discovered by security researcher Hyunwoo Kim, Dirty Frag belongs to the same bug class as Dirty Pipe and Copy Fail but exploits the fragment field of a different kernel data structure — the `algif_aead` cryptographic algorithm interface, introduced roughly nine years ago. The exploit chains two separate kernel flaws: the xfrm-ESP Page-Cache Write vulnerability and the RxRPC Page-Cache Write vulnerability, modifying protected system files in memory without authorization. Unlike race-condition exploits, Dirty Frag is a deterministic logic bug with a high success rate and no kernel panic on failure. The vulnerability has not yet received a CVE-ID, and no patches exist. Kim released full documentation and a PoC exploit after a third party broke the responsible disclosure embargo on May 7. Mitigation requires removing the vulnerable `esp4`, `esp6`, and `rxrpc` kernel modules — which breaks IPsec VPNs and AFS distributed filesystems. Affected distros include Ubuntu, RHEL, CentOS Stream, AlmaLinux, openSUSE Tumbleweed, and Fedora. This disclosure lands while Linux maintainers are still rolling out patches for Copy Fail (CVE-2026-31431), which CISA added to KEV last Friday with a May 15 remediation deadline. **[NEW] ClaudeBleed — Claude Chrome extension takeover via zero-permission malicious extensions, partial fix bypassable**

LayerX has disclosed "ClaudeBleed," a vulnerability in the Claude extension for Chrome that allows any installed extension — even one with zero special permissions — to take over the AI agent through remote prompt injection. The root cause is Claude's reliance on origin-based trust (`claude.ai`) rather than execution context validation, allowing any JavaScript running in the page to issue privileged commands. Attackers can weaponize Claude to exfiltrate data from Gmail, GitHub, and Google Drive; send emails; delete data; and share documents without authorization. Anthropic's partial fix adds security checks for "standard" mode extensions but does not address the fundamental trust model flaw — attackers can bypass the fix by switching their extension to "privileged" mode, which triggers no user notification or approval prompt. [[SecurityWeek](https://www.securityweek.com/vulnerability-in-claude-extension-for-chrome-exposes-ai-agent-to-takeover/)]

SentinelLabs has documented PCPJack, a new credential-stealing malware framework actively spreading through exposed cloud and container infrastructure. ## ⚠️ Vulnerabilities & Patches

**[NEW] Microsoft Semantic Kernel AI agent RCE — prompt injection achieves host-level code execution via two CVEs**

Microsoft's security research team has disclosed two critical vulnerabilities in Semantic Kernel, the company's open-source AI agent framework with 27,000+ GitHub stars. CVE-2026-26030 exploits unsafe string interpolation in the In-Memory Vector Store's filter function, where AI model-controlled parameters are passed directly into `eval()` with an insufficient AST blocklist — a single prompt injection bypasses all protections via Python class hierarchy traversal to achieve arbitrary shell command execution. CVE-2026-25592 is a sandbox escape in the SessionsPythonPlugin: an accidentally exposed `[KernelFunction]` attribute on `DownloadFileAsync` allows prompt injection to write attacker-controlled files to arbitrary host paths, including the Windows Startup folder for persistent RCE. Both vulnerabilities are patched (Python SDK 1.39.4, .NET SDK 1.71.0). Microsoft emphasizes that AI models are not security boundaries — any tool parameter the model can influence must be treated as attacker-controlled input. The company will publish follow-up research on similar vulnerabilities in third-party agent frameworks. [[Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/05/07/prompts-become-shells-rce-vulnerabilities-ai-agent-frameworks/)]

**[NEW] Ivanti EPMM zero-day (CVE-2026-6973) actively exploited — 850+ instances exposed online**

Patches are available in EPMM 12.6.1.1, 12.7.0.1, and 12.8.0.1. This brings CISA's total count of exploited Ivanti vulnerabilities to 33, with 12 also abused by ransomware operations. **[NEW] Rancher Fleet CVE-2026-41050 — multi-tenant isolation bypass yields full cluster-admin privileges**

SUSE Rancher has disclosed a critical vulnerability in Rancher Fleet, the GitOps tool for managing Kubernetes clusters at scale. CVE-2026-41050 breaks Fleet's core multi-tenant isolation by failing to enforce ServiceAccount impersonation across deployment pipelines. An attacker with basic git push access to a monitored repository can deploy a malicious Helm chart that extracts admin tokens from any namespace across all downstream clusters using the Helm `lookup` function — which executes with the highly privileged fleet-agent credentials rather than the restricted tenant account. A second bypass via `valuesFrom` in FleetFleet.yaml reads secrets with cluster-admin privileges. Stolen credentials (e.g., AWS IAM roles) enable lateral movement across corporate infrastructure. Affected versions: Fleet before 0.11.13, 0.12.14, 0.13.10, and 0.14.5; Rancher 2.10.11 and older. [[GBHackers](https://gbhackers.com/critical-vulnerability-in-rancher-fleet/)]

## 🛡️ Defense & Detection

**[NEW] Claude Code OAuth tokens stealable via stealthy MCP hijacking — Anthropic dismisses as "out of scope"**

Mitiga Labs has disclosed an attack chain that steals OAuth tokens from Claude Code through MCP (Model Context Protocol) configuration hijacking. The attack exploits plaintext OAuth token storage in `~/.claude.json` combined with npm lifecycle hooks: a malicious npm package registers a post-install hook that silently modifies MCP server configurations to route all connections through an attacker-controlled proxy. The hook bypasses Claude Code's trust dialog by pre-setting trust flags, making the interception invisible to the user. Stolen tokens provide MFA-bypassing access to all connected SaaS tools. The attack is self-healing — the hook re-applies on subsequent loads, surviving token rotation. Mitiga reported the findings to Anthropic on April 10; Anthropic dismissed them as "out of scope" on April 12, reasoning that the user has already consented to what might happen. Organizations using Claude Code should monitor for configuration changes and MCP server URL modifications. [[SecurityWeek](https://www.securityweek.com/claude-code-oauth-tokens-can-be-stolen-through-stealthy-mcp-hijacking/)]

**[NEW] TCLBanker banking trojan spreads via WhatsApp and Outlook worms — trojanized Logitech installer**

Elastic Security Labs has documented TCLBanker, a new Brazilian banking trojan targeting 59 banking, fintech, and cryptocurrency platforms. The malware arrives via a trojanized MSI installer for Logitech AI Prompt Builder and loads through DLL side-loading within the legitimate application context. TCLBanker features worm modules for self-propagation: it hijacks WhatsApp Web sessions via Chromium IndexedDB data and Outlook via COM automation to send phishing messages from the victim's accounts. The banking module monitors browser address bars using Windows UI Automation APIs, establishing WebSocket sessions with C2 when targeted sites are visited for live screen streaming, keylogging, clipboard hijacking, and overlay-based credential theft. The malware actively hunts for analysis tools (x64dbg, IDA, dnSpy, Frida) and kills Task Manager during active sessions. Code artifacts suggest AI-assisted development. [[BleepingComputer](https://www.bleepingcomputer.com/news/security/new-tclbanker-malware-self-spreads-over-whatsapp-and-outlook/); [GBHackers](https://gbhackers.com/signed-logitech-installer-abused/)]
