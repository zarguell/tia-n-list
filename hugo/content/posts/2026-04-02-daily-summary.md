# Daily Threat Intel Digest - April 2, 2026

## 🔴 Critical Threats & Active Exploitation

**[UPDATE] Axios npm supply chain compromise attributed to North Korean Sapphire Sleet (UNC1069)**
A North Korean state-sponsored threat actor compromised the maintainer account for `axios`—a JavaScript HTTP client with over 100 million weekly downloads—publishing malicious versions `1.14.1` and `0.30.4` on March 31. The packages contained a hidden dependency (`plain-crypto-js@4.2.1`) that executed a post-install script to deliver WAVESHAPER.V2, a cross-platform RAT targeting macOS, Windows, and Linux systems. Google Threat Intelligence Group (GTIG) has attributed the attack to UNC1069 (also tracked as Sapphire Sleet, STARDUST CHOLLIMA, and CryptoCore), a financially motivated group focused on cryptocurrency and blockchain theft since at least 2018 [[Tenable FAQ](https://www.tenable.com/blog/faq-about-the-axios-npm-supply-chain-attack-by-north-korea-nexus-threat-actor-unc1069); [Microsoft Blog](https://www.microsoft.com/en-us/security/blog/2026/04/01/mitigating-the-axios-npm-supply-chain-compromise/)]. 

**What’s new:** Microsoft has released a deep technical breakdown of the attack chain and comprehensive mitigation guidance. The threat actor compromised the npm account `@jasonsaayman` using a long-lived classic access token, changing the associated email to an attacker-controlled ProtonMail address. The malicious packages were live on the npm registry for approximately three hours before automated detection by Socket.dev led to their removal. GTIG warns that "hundreds of thousands of stolen secrets could potentially be circulating" as a result of this attack. Defenders should immediately audit CI/CD pipelines for the presence of `plain-crypto-js` or the compromised Axios versions, block C2 infrastructure (`sfrclak[.]com` and `142.11.206[.]73` on port 8000), and treat affected endpoints as fully compromised, rotating all secrets and rebuilding from clean snapshots [[Microsoft Blog](https://www.microsoft.com/en-us/security/blog/2026/04/01/mitigating-the-axios-npm-supply-chain-compromise/); [GBHackers](https://gbhackers.com/axios-npm-supply-chain/); [CrowdStrike Blog](https://www.crowdstrike.com/en-us/blog/stardust-chollima-likely-compromises-axios-npm-package/)].

**[NEW] TrueConf zero-day exploited to push malicious updates to government endpoints**
A Chinese-nexus threat actor is exploiting a zero-day vulnerability (CVE-2026-3502) in TrueConf video conferencing servers to distribute malicious software updates to all connected clients. By compromising a single centrally managed government TrueConf server, attackers replaced legitimate update packages with arbitrary executables that were automatically deployed to connected endpoints. The "TrueChaos" operation targets Southeast Asian government entities and uses DLL sideloading, UAC bypasses, and eventually deploys the open-source Havoc C2 framework for persistent remote access. The flaw affects TrueConf versions 8.1.0 through 8.5.2 and was patched in version 8.5.3 following disclosure by Check Point Research [[BleepingComputer](https://www.bleepingcomputer.com/news/security/hackers-exploit-trueconf-zero-day-to-push-malicious-software-updates/)].

**[UPDATE] F5 BIG-IP APM RCE - over 14,000 instances remain exposed**
Following F5's reclassification of CVE-2025-53521 from a denial-of-service flaw to a critical remote code execution vulnerability, Shadowserver reports that more than 14,000 BIG-IP APM instances remain exposed on the internet. Attackers are actively exploiting this flaw on unpatched systems with access policies configured on a virtual server. CISA has ordered federal agencies to secure their BIG-IP systems and added the vulnerability to its Known Exploited Vulnerabilities catalog. F5 has published indicators of compromise (IOCs) and strongly advises affected organizations to rebuild compromised systems from scratch, as persistent malware can survive in UCS configuration backups [[BleepingComputer](https://www.bleepingcomputer.com/news/security/over-14-000-f5-big-ip-apm-instances-still-exposed-to-rce-attacks/)].

**[NEW] Critical Cisco IMC authentication bypass enables persistent server takeover**
Cisco has disclosed a maximum-severity vulnerability (CVE-2026-20093, CVSS 9.8) in its Integrated Management Controller (IMC) software that allows unauthenticated remote attackers to overwrite administrative passwords and gain full control of affected hardware. Because Cisco IMC operates as an out-of-band management controller independent of the host OS, an attacker's access persists through host reboots and re-imaging. The flaw affects a wide range of enterprise hardware, including UCS C-Series M5/M6 Rack Servers, Catalyst 8300 Series Edge platforms, 5000 Series ENCS, and numerous pre-configured network appliances like Secure Firewall Management Center and Catalyst Center Appliances. Cisco has confirmed no workarounds exist, and patches must be applied immediately [[Cyberpress](https://cyberpress.org/cisco-integrated-management-controller-flaw/); [GBHackers](https://gbhackers.com/cisco-warns-of-critical-imc-vulnerability/)].

**[NEW] Critical Cisco Smart Software Manager flaw allowed root-level command execution**
In a second critical Cisco advisory, a vulnerability (CVE-2026-20160, CVSS 9.8) in the Smart Software Manager On-Prem (SSM On-Prem) platform enables unauthenticated remote attackers to execute arbitrary commands with root privileges on the underlying operating system. This product manages software licensing across enterprise environments, meaning a compromise could facilitate lateral movement and persistent access to downstream managed network infrastructure. Cisco has released patched software versions and administrators should prioritize immediate upgrades [[GBHackers](https://gbhackers.com/cisco-smart-software-manager-flaw/)].

## 🎯 Threat Actor Activity & Campaigns

**[NEW] Threat actors weaponize Anthropic Claude Code leak with Vidar and GhostSocks malware**
Following the accidental exposure of Anthropic's Claude Code source code via a public npm package, threat actors are actively creating trojanized GitHub repositories masquerading as the "leaked source." Zscaler ThreatLabz discovered a highly-ranked malicious repository distributing a Rust-based dropper that deploys Vidar v18.7 infostealer and GhostSocks proxy malware. The lure is particularly effective because it promises "unlocked" enterprise features with no message limits. Defenders should warn development teams that no version of the leaked Claude Code available on GitHub is safe to download, build, or run. Zero Trust segmentation of developer workstations is recommended to limit potential blast radius from trojanized AI tools [[Zscaler ThreatLabz](https://www.zscaler.com/blogs/security-research/anthropic-claude-code-leak)].

**[UPDATE] TA416 expands European espionage campaign to Middle Eastern diplomatic targets**
The China-aligned threat actor TA416 has resumed large-scale espionage operations against European governments following a two-year lull, and has now expanded targeting to include Middle Eastern diplomatic entities. The group continues to evolve its malware delivery chains, combining web bug reconnaissance for target validation with customized PlugX backdoor deployments. Organizations in the government and diplomatic sectors should heighten monitoring for spear-phishing indicators and review endpoint detection rules for PlugX variants [[GBHackers](https://gbhackers.com/ta416-broadens-europe/)].

**[NEW] North Korean IT worker laptop farm exposed with 40+ devices logged into corporate networks**
Cybersecurity firm Nisos uncovered a sophisticated North Korean employment fraud scheme involving a remote worker using a stolen Florida identity to apply for a Lead AI Architect position. Investigation of a corporate laptop shipped to the operative revealed a laptop farm containing approximately 40 devices, each logged into different corporate networks under fake employee names. Operatives used PiKVM hardware devices for stealthy remote control that evades endpoint detection, Astrill VPN for location masking, and Tailscale mesh VPN for secure command and control. Organizations hiring remote workers should implement rigorous identity verification processes and watch for indicators like conflicting resume details and VPN-masked IP addresses [[Cyberpress](https://cyberpress.org/north-korean-job-scam/)].

**[NEW] TheGentlemen ransomware toolkit exposed on misconfigured server**
Security researchers discovered an open directory on a bulletproof hosting provider (Proton66) containing the complete toolkit of "TheGentlemen" ransomware group. The leak includes over 120 files covering every stage of attack—from network scanning and privilege escalation to defense evasion and final ransomware deployment. Mimikatz output logs confirm that the toolkit was actively used against real victims. Notably, the toolkit includes Ngrok authentication tokens for hidden remote access tunnels and automated batch scripts that disable Windows Defender, clear event logs, and delete system backups prior to encryption [[Cyberpress](https://cyberpress.org/exposed-server-leaks-toolkit/)].

**[NEW] EvilTokens phishing kit automates Microsoft device code attacks for BEC operators**
A new phishing-as-a-service (PhaaS) platform called EvilTokens is enabling widespread business email compromise (BEC) campaigns by exploiting the OAuth 2.0 device authorization flow. The kit provides ready-made phishing templates impersonating DocuSign, Adobe Acrobat, and SharePoint, directing victims to authorize a malicious device via the legitimate Microsoft login page. Attackers receive persistent refresh tokens granting access to email, Teams, and files. Sekoia observed campaigns with global reach, primarily targeting the United States, Canada, France, and Australia, with plans to expand to Gmail and Okta targets [[BleepingComputer](https://www.bleepingcomputer.com/news/security/new-eviltokens-service-fuels-microsoft-device-code-phishing-attacks/)].

## ⚠️ Vulnerabilities & Patches

**[NEW] Critical Symantec DLP Agent privilege escalation from hardcoded developer path**
A vulnerability in the Symantec Data Loss Prevention (DLP) Agent for Windows (CVE-2026-3991, CVSS 7.8) allows low-privileged attackers to escalate to SYSTEM-level privileges without special permissions. The flaw stems from a hardcoded developer build path (`C:\VontuDev\workDir\openssl\output\x64\Release\SSL\openssl.cnf`) accidentally left in the OpenSSL configuration. Because this path does not exist on standard Windows installations, any authenticated user can create the directory structure and plant a malicious configuration file and DLL that the privileged `edpa.exe` process loads at startup. The injected code inherits SYSTEM privileges, enabling attackers to bypass endpoint detection. Broadcom has released patches in versions 16.1 MP2 and 25.1 MP1 [[Cyberpress](https://cyberpress.org/vulnerability-in-symantec-dlp/); [GBHackers](https://gbhackers.com/symantec-dlp-agent-flaw/)].

**[NEW] Critical PX4 Autopilot flaw enables complete drone hijack via MAVLink**
CISA has issued a high-priority alert for a critical vulnerability (CVE-2026-1579, CVSS 9.8) in the PX4 Autopilot open-source flight control software used globally for drones and unmanned aerial vehicles. The vulnerability stems from missing cryptographic authentication in the MAVLink communication protocol, allowing unauthenticated attackers with network access to send arbitrary commands—including the `SERIAL_CONTROL` command that provides interactive shell access to the flight controller. The flaw impacts critical infrastructure sectors including transportation, emergency services, and defense. Operators should immediately enable MAVLink 2.0 message signing for all non-USB communication links [[Cyberpress](https://cyberpress.org/critical-px4-autopilot-flaw-lets-hackers-take-control-of-drones/); [GBHackers](https://gbhackers.com/critical-px4-autopilot-vulnerability/)].

**[NEW] Drupal SAML SSO module critical authentication bypass**
Drupal has released a security advisory for a critical authentication bypass vulnerability in the SAML SSO - Service Provider module (versions prior to 3.1.4). This flaw could allow attackers to bypass authentication controls on Drupal sites relying on SAML-based single sign-on, potentially exposing sensitive content and administrative functions. Organizations using Drupal with SAML authentication should apply the update immediately [[Canadian Centre for Cyber Security](https://cyber.gc.ca/en/alerts-advisories/drupal-security-advisory-av26-308)].

## 🛡️ Defense & Detection

**[NEW] GhostSocks malware converts endpoints into residential proxies for ransomware operators**
Darktrace research details how GhostSocks malware covertly converts compromised devices into residential SOCKS5 proxies wrapped in TLS to evade network monitoring. Recent variants establish persistence via registry run keys and operate as a backdoor capable of deploying secondary payloads. The malware has been observed working in tandem with Lumma Stealer and has reportedly been leveraged by the Black Basta ransomware group to maintain long-term, covert access to victim networks. In a late 2025 incident, a compromised device downloaded the malware via a file named `Renewable.exe` within two minutes of contacting Lumma C2 infrastructure. Network defenders should hunt for TLS connections to `retreaw[.]click` and `www.lbfs[.]site` [[Cyberpress](https://cyberpress.org/ghostsocks-builds-proxy-network/)].

**[NEW] RoadK1ll: Lightweight Node.js pivoting implant uses custom protocol for lateral movement**
Blackpoint's Response Operations Center has documented RoadK1ll, a Node.js-based implant that converts a single compromised machine into a network relay. The malware uses a custom communication protocol via WebSockets, enabling attackers to initiate TCP connections to internal destinations that bypass perimeter security controls. Lacking traditional persistence mechanisms, the implant instead relies on an automatic reconnection loop to maintain access. Incident response teams should look for WebSocket connections using custom message framing structures and review internal traffic originating from unexpected Node.js processes [[Cyberpress](https://cyberpress.org/roadk1ll-enables-network-pivoting/)].
