# Daily Threat Intel Digest - April 30, 2026

## 🔴 Critical Threats & Active Exploitation

**[NEW] Linux kernel "Copy Fail" zero-day (CVE-2026-31431) grants unprivileged root access on all major distros since 2017 — and escapes containers**

A logic flaw in the Linux kernel's `authencesn` cryptographic template, reachable via the `AF_ALG` socket interface and `splice()` system call, enables any unprivileged local user to gain full root access with a single 732-byte Python script using only standard library modules. Dubbed "Copy Fail," the vulnerability originates from a 2017 in-place optimization in `algif_aead.c` that causes page cache pages to be placed into a writable destination scatterlist, allowing a controlled 4-byte write into in-memory setuid binaries like `/usr/bin/su`. Because the kernel never marks corrupted pages as dirty for writeback, on-disk files remain untouched — defeating checksum-based integrity tools entirely. Confirmed exploitation on Ubuntu 24.04 LTS, Amazon Linux 2023, RHEL 10.3, and SUSE 16. Critically, Copy Fail functions as a **Kubernetes container escape primitive** because the page cache is shared across all processes on a host, including across container boundaries. A compromised container can corrupt setuid binaries visible to other containers and the host kernel. Three independent code changes from 2011, 2015, and 2017 converged to create this condition with no developer connecting their intersection for nearly a decade. Patched in the kernel on April 1 (CVE assigned April 22, public disclosure April 29). Immediate mitigation: `echo "install algif_aead /bin/false" > /etc/modprobe.d/disable-algif-aead.conf && rmmod algif_aead`. [[Cyber Security News](https://cyberpress.org/linux-kernel-0-day-copy-fail/); [SecurityWeek](https://www.securityweek.com/copy-fail-logic-flaw-in-linux-kernel-enables-system-takeover/)]

**[UPDATE] cPanel/WHM auth bypass (CVE-2026-41940) was exploited as a zero-day since at least February 23**

Managed hosting provider KnownHost reports that CVE-2026-41940 was actively exploited in the wild prior to public disclosure, with targeted zero-day attacks potentially starting as early as February 23, 2026 — over two months before cPanel's April 28 emergency patch. Security firm watchTowr has published a technical analysis and proof-of-concept exploit demonstrating the CRLF injection technique that allows unauthenticated attackers to inject `user=root` into session files, bypassing encryption on the `whostmgrsession` cookie. Shodan enumerates approximately 1.5 million cPanel instances exposed to the internet. The vulnerability affects all versions after 11.40; fix requires manual execution of `/usr/local/cpanel/scripts/upcp` as automatic update channels do not deliver it. [[Rapid7](https://www.rapid7.com/blog/post/etr-cve-2026-41940-cpanel-whm-authentication-bypass); [SecurityWeek](https://www.securityweek.com/critical-cpanel-whm-vulnerability-exploited-as-zero-day-for-months/); [BleepingComputer](https://www.bleepingcomputer.com/news/security/cpanel-whm-emergency-update-fixes-critical-auth-bypass-bug/)]

## 🎯 Threat Actor Activity & Campaigns

**[NEW] North Korea's Famous Chollima uses LLM-optimized supply chain attack ("PromptMink") delivered via Claude AI-generated commits**

North Korean state-sponsored group Famous Chollima has been identified as the operator behind "PromptMink," a multi-layered malware campaign that exploits AI coding assistants to introduce malicious dependencies into open-source projects. In a February 2026 commit to the `openpaw-graveyard` autonomous crypto trading project, Anthropic's Claude Opus model added a malicious package `@validate-sdk/v2` as a dependency — disguised as a data validation tool but actually siphoning cryptocurrency wallet credentials and deploying persistent SSH keys. The attack uses a two-layer structure: clean "bait" packages like `@solana-launchpad/sdk` that silently import the malicious payload from second-layer packages, allowing the attacker to swap compromised packages without losing reputation. ReversingLabs attributes the campaign to Famous Chollima and describes it as pioneering "LLM Optimization (LLMO)" abuse — writing convincing documentation that tricks AI coding agents into recommending and installing malicious packages. [[Cyber Security News](https://cyberpress.org/claude-commit-targets-crypto/); [ReversingLabs](https://blog.reversinglabs.com/blog/promptmink-north-korean-supply-chain-attack-targets-crypto-developers)]

## ⚠️ Vulnerabilities & Patches

**[NEW] ProFTPD pre-auth SQL injection (CVE-2026-42167, CVSS 8.1) enables RCE, auth bypass, and privilege escalation on 162K+ exposed instances**

A critical SQL injection vulnerability in ProFTPD's `mod_sql` extension allows pre-authentication remote code execution, authentication bypass, and privilege escalation. The flaw stems from how `mod_sql`-based logging handles `%` expansions (e.g., `%U` for username) in `SQLNamedQuery` directives — values beginning and ending with single quotes bypass SQL escaping entirely. An attacker can inject arbitrary SQL via a crafted `USER` command before ever authenticating. When ProFTPD connects to PostgreSQL with superuser privileges, the injection enables full RCE. ZeroPath estimates at least 1% of publicly accessible ProFTPD instances are vulnerable to pre-auth attacks. ProFTPD is bundled with cPanel, Plesk, DirectAdmin, ISPConfig, and Webmin, making this a significant supply-chain risk for shared hosting environments. Patch available in ProFTPD 1.3.9a (released April 27). [[Cyber Security News](https://cyberpress.org/proftpd-sql-injection-flaw/)]

**[NEW] Qinglong task scheduler auth bypass (CVE-2026-3965, CVE-2026-4047) actively exploited for cryptomining since February**

Two authentication bypass vulnerabilities in Qinglong, a popular open-source task scheduling platform with 19,000+ GitHub stars, have been actively exploited since at least February 7, 2026 — weeks before public disclosure. The flaws stem from mismatches between Express.js routing behavior and the security middleware: CVE-2026-3965 exposes protected admin endpoints through a misconfigured rewrite rule mapping `/open/*` to `/api/*`, while CVE-2026-4047 exploits case-sensitive authentication checks against case-insensitive routing. Attackers deploy a cryptominer disguised as a legitimate Java garbage collection process (`.fullgc`) that consumes 85-100% of CPU. Qinglong's initial mitigation (PR #2924) was insufficient; the effective fix came in PR #2941. [[BleepingComputer](https://www.bleepingcomputer.com/news/security/hackers-exploit-rce-flaws-in-qinglong-task-scheduler-for-cryptomining/); [Cyber Security News](https://cyberpress.org/qinglong-task-scheduler-rce-flaws/)]

**[NEW] SonicWall SonicOS three vulnerabilities affect all firewall generations**

SonicWall has disclosed three vulnerabilities (CVE-2026-0204, CVE-2026-0205, CVE-2026-0206) spanning its entire Gen6, Gen7, and Gen8 firewall product lines. The most critical, CVE-2026-0204 (CVSS 8.0), is a weak authentication flaw allowing adjacent-network attackers to bypass access controls on the management interface. CVE-2026-0205 (CVSS 6.8) is a post-auth path traversal enabling file access, and CVE-2026-0206 (CVSS 4.9) is a post-auth stack-based buffer overflow in the SSL-VPN component that crashes the firewall entirely, causing full denial-of-service. Discovered by CrowdStrike's Advanced Research Team. SonicWall urges administrators to immediately disable HTTP/HTTPS-based firewall management and SSL-VPN access until patches are applied. [[Cyber Security News](https://cyberpress.org/sonicwall-sonicos-access-controls-crash-firewall/)]

## 🛡️ Defense & Detection

**[NEW] WordPress "Quick Page/Post Redirect" plugin backdoor dormant on 70,000+ sites**

A backdoor added to the Quick Page/Post Redirect plugin (70,000+ active installations) in 2020-2021 silently persisted for five years before discovery. Versions 5.2.1 and 5.2.2 included a hidden self-update mechanism pointing to `anadnet[.]com`, which pushed a tampered 5.2.3 build containing a passive backdoor hooked into `the_content`. The backdoor, active only for logged-out users to evade admin detection, was used for SEO spam operations — effectively renting Google ranking on 70,000 websites. While the malicious self-updater was removed from subsequent WordPress.org versions, the external command-and-control domain remains active, and the arbitrary code execution capability is still present on affected installs. WordPress.org has temporarily pulled the plugin. Organizations should immediately uninstall the plugin and replace it with a clean copy of version 5.2.4. [[BleepingComputer](https://www.bleepingcomputer.com/news/security/popular-wordpress-redirect-plugin-hid-dormant-backdoor-for-years/)]
