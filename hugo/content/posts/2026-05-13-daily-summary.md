## 🔴 Critical Threats & Active Exploitation

**[NEW] West Pharmaceutical Services hit by disruptive ransomware attack — global operations halted**

West Pharmaceutical Services, a $10B Pennsylvania-based manufacturer of injectable drug delivery systems, filed an SEC 8-K on May 7 disclosing a material ransomware attack. Intrusion was detected on May 4, with threat actors exfiltrating data and encrypting systems. The company proactively took all systems offline globally for containment, disrupting shipping, receiving, and manufacturing operations across all sites. Core enterprise systems have since been restored and critical processes restarted at some facilities, but full recovery timeline remains undetermined. Palo Alto Networks' Unit 42 has been engaged for incident response. No ransomware group has claimed responsibility as of May 13. West Pharma, which reported $3.5 billion in 2025 revenue, is a critical link in the pharmaceutical supply chain — injection-molded components from West are used in the majority of injectable drugs administered worldwide. The attack follows the broader pattern of healthcare/pharma sector targeting noted throughout 2026. [[The Record](https://therecord.media/west-pharmaceutical-warns-of-ransomware-attack-impacting-operations); [SEC 8-K Filing](https://www.sec.gov/Archives/edgar/data/105770/000010577026000068/wst-20260507.htm); [SecurityWeek](https://www.securityweek.com/west-pharmaceutical-services-hit-by-disruptive-ransomware-attack/)]

**[NEW] Cline AI coding assistant — critical unpatched RCE via WebSocket hijack (CVE-2026-44211, CVSS 9.3)**

A critical zero-authentication vulnerability in the `kanban` npm package bundled with the Cline AI coding assistant allows any malicious website to silently hijack a developer's AI agent terminal, leak workspace data in real-time, and achieve full remote code execution. Tracked as CVE-2026-44211 (CVSS 9.3), the flaw stems from the local WebSocket server on `127.0.0.1:3484` performing zero Origin header validation and requiring no authentication on any of its three endpoints. An attacker-controlled webpage can connect to `/api/runtime/ws` to receive filesystem paths, git branches, task descriptions, and live AI chat; escalate to `/api/terminal/io` to inject raw shell commands into the agent's PTY (achieving RCE); and use `/api/terminal/control` to kill running sessions. A working PoC has been demonstrated across macOS, Linux, and Windows on Firefox, Chrome, and Arc. **No patched version exists as of disclosure.** All Cline versions before v2.13.0 are vulnerable. Developers running Cline with Kanban active should treat any visited webpage as a potential attack vector until a fix ships. [[GitHub Advisory GHSA-5c57-rqjx-35g2](https://github.com/cline/cline/security/advisories/GHSA-5c57-rqjx-35g2); [Cyber Defense](https://thecybrdef.com/critical-cline-kanban-remote-code-execution-vulnerability/)]

**[NEW] Foxconn confirms cyberattack on North American factories — Nitrogen ransomware claims 8TB theft [TLDR InfoSec; Reddit r/cybersecurity]**

Foxconn, the world's largest contract electronics manufacturer ($258.3B 2025 revenue), confirmed a cyberattack affecting multiple North American factories including its Wisconsin AI hub and Houston facility. The Nitrogen ransomware gang claimed responsibility, asserting theft of 8TB / 11M+ files including schematics for Apple, NVIDIA, Dell, Google, and Intel products. Nitrogen, built on Conti's builder, has a reportedly broken decryptor per Coveware (Feb 2026), meaning paying ransom offers no recovery guarantee. ## 🎯 Threat Actor Activity & Campaigns

**[UPDATE] Mini Shai-Hulud supply chain attack escalates — hundreds of npm packages compromised with signed provenance bypass, RubyGems suspends registrations**

TeamPCP's supply chain campaign, first documented in yesterday's digest as compromising 84 TanStack npm packages, has expanded dramatically. New analysis by CyberScoop reveals the attack used an "orphaned commit" technique — code pushed to a repository fork without a corresponding branch — to exploit overly broad permissions in GitHub Actions workflows. The malicious updates bypassed two-factor authentication and carried cryptographically valid provenance signatures, exposing a fundamental blind spot: current defenses verify where an update comes from, not whether the code inside is safe. The malware uses the Bun runtime to steal AWS, GCP, Kubernetes, and HashiCorp Vault credentials, embeds persistence hooks in VS Code and Claude Code configuration directories, and exfiltrates via the Session anonymous messaging network. It also self-propagates as a worm, spoofing activity to appear as automated commits from the Anthropic Claude bot. Separately, RubyGems has suspended new package registrations after hundreds of malicious packages were discovered flooding the registry — part of the same coordinated campaign targeting all major package ecosystems. Socket CEO Feross Aboukhadijeh noted: "You can pull a package from the registry, but you cannot automatically pull back the credentials it may have already stolen." [[CyberScoop](https://cyberscoop.com/mini-shai-hulud-supply-chain-malware-attack/); [BleepingComputer](https://www.bleepingcomputer.com/news/security/shai-hulud-attack-ships-signed-malicious-tanstack-mistral-npm-packages/); [SecurityWeek](https://www.securityweek.com/hundreds-of-malicious-packages-force-rubygems-to-suspend-registrations/)]

**[NEW] ModeloRAT delivered via hijacked Microsoft Teams accounts — KongTuke group evolves delivery methods**

The threat group KongTuke has shifted from email-based phishing to directly hijacking Microsoft Teams accounts to deliver an evolved variant of ModeloRAT. Attackers pose as internal IT helpdesk staff over Teams, instructing victims to run obfuscated PowerShell that downloads a Dropbox-hosted ZIP containing a portable Python environment and malicious scripts. The new ModeloRAT variant splits execution into reconnaissance and C2 connection components, achieving zero detections on public malware scanning engines at time of discovery. Persistence is maintained via hidden registry keys and scheduled tasks with randomized names. Organizations should restrict Teams external access settings, block or alert on Dropbox downloads from corporate endpoints, and hunt for unauthorized ZIP extraction in hidden application directories. [[Cyber Security News](https://cyberpress.org/hijacked-teams-deliver-modelorat/)]

## ⚠️ Vulnerabilities & Patches

**[NEW] Microsoft May 2026 Patch Tuesday — 137 vulnerabilities, 13 critical, first zero-day-free Patch Tuesday since June 2024**

Microsoft addressed 137 vulnerabilities across its product ecosystem with no zero-days actively exploited or publicly disclosed — the first such "clean" Patch Tuesday in nearly two years. Notable critical fixes include CVE-2026-41089 (CVSS 9.8): a stack-based buffer overflow in Windows Netlogon allowing unauthenticated RCE on domain controllers with no user interaction required; CVE-2026-41096: critical RCE in the Windows DNS client; CVE-2026-41103 (CVSS 9.1): elevation of privilege in the Microsoft SSO Plugin for Jira & Confluence allowing forged credential bypass of Entra ID authentication; and four critical Microsoft Word RCE vulnerabilities (CVE-2026-40361/40364/40366/40367, all CVSS 8.4) exploitable via the Preview Pane. This month's Patch Tuesday is notably the first to reflect the impact of Anthropic's "Project Glasswing" AI vulnerability discovery initiative — Krebs on Security reports that Apple shipped fixes for 52 iOS vulnerabilities (triple its average), Mozilla patched 271 in Firefox 150, and Oracle addressed 450+ including 300+ remotely exploitable flaws, all attributed to AI-assisted discovery. **[NEW] Exim "Dead.Letter" — unauthenticated RCE in internet-facing mail servers (CVE-2026-45185) [TLDR InfoSec]**

XBOW security researchers have disclosed CVE-2026-45185, an unauthenticated use-after-free in Exim's BDAT body parsing path when built with GnuTLS. During TLS shutdown, a nested BDAT receive wrapper can still process bytes and write a newline into freed memory, providing a path from small heap corruption to code execution. Affects Exim 4.97–4.99.2 with STARTTLS and CHUNKING enabled. XBOW describes it as "one of the highest-caliber bugs" found in Exim. Fixed in version 4.99.3. Exim runs approximately 50-60% of the internet's mail servers — administrators should patch immediately and review logs for unusual STARTTLS/BDAT session patterns, crashes, or paniclog entries. [[GridinSoft](https://blog.gridinsoft.com/exim-cve-2026-45185-dead-letter-rce/); [Debian DSA-6265-1](https://linuxsecurity.com/advisories/debian/debian-dsa-6265-1-exim4)]

**[NEW] Ollama "Bleeding Llama" — critical heap memory leak in AI inference server (CVE-2026-7482, CVSS 9.1) [TLDR InfoSec]**

Cyera Research has disclosed CVE-2026-7482, a critical heap out-of-bounds read in Ollama's GGUF model loader affecting ~300,000 exposed servers on the public internet. An unauthenticated attacker uploads a crafted model via `/api/create` with an inflated tensor shape, triggers an OOB read during quantization, and exfiltrates the entire process heap via `/api/push` — all in 3 API calls with no authentication required in the default configuration (listens on 0.0.0.0). Leaked data includes user prompts, system prompts, API keys, environment variables, and all heap memory contents. The F16→F32 conversion is lossless, preserving heap data exactly. Ollama (170K+ GitHub stars, 100M+ Docker Hub downloads) is widely deployed as enterprise AI "chat" infrastructure. Fixed in v0.17.1. Organizations running Ollama should update immediately, restrict network access, and rotate any API keys that were present in server memory. [[Cyera Research](https://www.cyera.com/research/bleeding-llama-critical-unauthenticated-memory-leak-in-ollama)]

**[NEW] Fortinet critical RCE — CVE-2026-44277 (FortiAuthenticator) and CVE-2026-26083 (FortiSandbox) allow unauthenticated code execution**

Fortinet has released security updates for two critical vulnerabilities. CVE-2026-44277 is an improper access control flaw in FortiAuthenticator (IAM solution) allowing unauthenticated attackers to execute arbitrary code via crafted HTTP requests — patched in versions 6.5.7, 6.6.9, and 8.0.3 (Cloud version unaffected). CVE-2026-26083 is a missing authorization vulnerability in FortiSandbox, FortiSandbox Cloud, and FortiSandbox PaaS enabling unauthenticated RCE via HTTP requests. Neither flaw is currently tagged as exploited in the wild, but Fortinet vulnerabilities have been exploited as zero-days 24 times per CISA's KEV catalog, with 13 also abused by ransomware. SAP's May 2026 security update addresses 15 vulnerabilities across its ecosystem. **[NEW] Android RCE via Wireless Debugging (CVE-2026-0073) — adjacent-network shell access with no user interaction**

Google has disclosed CVE-2026-0073, a critical authentication bypass in Android's ADB daemon (adbd) TLS certificate verification path. An adjacent attacker on the same Wi-Fi network can bypass the Wireless Debugging pairing requirement and obtain a remote shell as the Android shell user with no user interaction — provided the device has been paired with at least one device in the past. A public PoC exploit is available and has been successfully tested against Android 13-16 devices, tablets, smartwatches, and Google Chromecast TV. Google Pixel devices were not successfully exploited in testing. Patched in security level 2026-05-01. The practical risk is highest for developers, device farms, repair environments, and BYOD engineering fleets that enable Wireless Debugging. [[Mobile Hacker](https://www.mobile-hacker.com/2026/05/12/android-rce-via-wireless-debugging-from-network-access-to-shell/); [Malware.News](https://malware.news/t/android-rce-via-wireless-debugging-from-network-access-to-shell/106929#post_1)]

## 🛡️ Defense & Detection

**[NEW] Vercel AI tools abused for mass phishing production — GenAI compresses entire phishing kit lifecycle to text prompts**

Cofense documents a surge in campaigns using Vercel's v0[.]dev GenAI tool to mass-produce highly convincing phishing pages. Attackers generate spoofed Microsoft, Spotify, Facebook, and corporate career login pages via simple text prompts, deploy them instantly on Vercel's hosting, and integrate with Telegram bots for real-time credential exfiltration. The AI generates slightly different outputs per prompt, aiding evasion. Campaigns span fake job recruitment (Adidas, Nike, Ferrari), Microsoft credential harvesting, and Spotify payment capture. Vercel's free tier eliminates infrastructure costs. The broader trend: generative AI is transforming phishing from a technically demanding activity into an accessible, scalable operation requiring only basic knowledge and free tools. [[GBHackers](https://gbhackers.com/threat-actors-abuse-vercel-ai-tools/)]

**[NEW] Signal adds security warnings for social engineering and phishing — responds to Russian state-sponsored Linked Device attacks**

Signal has introduced in-app confirmations, warning messages, and enhanced safety tips to combat phishing and social engineering attacks targeting high-profile users. New protections include "Name not verified" and "No groups in common" labels on direct messages, confirmation prompts for new message requests with reminders that Signal never asks for registration codes, and explicit warnings about fake "Signal Support" contacts. The update responds directly to Russian state-sponsored campaigns — highlighted by the FBI, Dutch government, and German authorities — that abuse the Linked Device feature by convincing targets to scan QR codes or share verification codes, granting full account access. [[BleepingComputer](https://www.bleepingcomputer.com/news/security/signal-adds-security-warnings-for-social-engineering-phishing-attacks/)]
